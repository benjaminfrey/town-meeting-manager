# Session 24.01 — Maine GeoLibrary Data Import Pipeline

**Phase:** 3 — Scale | **Block:** 24 — Parcel Data Integration
**Dependencies:** 01.05
**Estimated tasks:** 11

---

## Description

Build the data import pipeline for Maine GeoLibrary parcel and E911 address data. This session creates PostGIS-enabled database tables, the import service that queries the ArcGIS REST API with pagination, E911 address import, coordinate system handling, automated monthly data refresh checks, an admin import dashboard, and manual parcel entry for towns without GeoLibrary coverage. The imported parcel data enables map display, abutters lists, and proximity notifications in subsequent sessions.

## Tasks

1. Create parcel import service in packages/api
2. Create PostGIS-enabled PARCEL table: id, town_id, state_id, map_bk_lot, geometry (PostGIS polygon), prop_loc, town, county, geocode, owner1, owner2, owner_address, land_val, bldg_val, land_use, land_use_desc, lot_size, source (geolibrary/manual), imported_at, geolibrary_updated_at
3. Create E911_ADDRESS table: id, town_id, state_id, map_bk_lot, address, street_name, address_number, latitude, longitude, geometry (PostGIS point), zipcode, building, place_type, imported_at
4. Create import job: query Maine GeoLibrary ArcGIS REST API for Maine_Parcels_Merged/FeatureServer/1 with where=TOWN='TOWNNAME'&f=geojson, handle pagination (2000 records per request via resultOffset)
5. Create E911 import job: query Addresses/FeatureServer/0 with same per-town approach
6. Handle coordinate system: request f=geojson for automatic reprojection to WGS 84 (EPSG:4326), test at integration time
7. Store town's GEOCODE from GeoLibrary during import for reliable matching
8. Create automated monthly check: compare record counts and timestamps against last import, import if changes detected, notify admin
9. Create admin import dashboard: last refresh date per town, record counts, "Refresh Now" button
10. Create manual parcel entry for towns without GeoLibrary data (MAP_BK_LOT, address, owner -- no geometry, flagged as manual)
11. Verify: import parcels for a test town, E911 addresses import, PostGIS geometry stores correctly, monthly check logic works

TASK 2: Create PostGIS-enabled PARCEL table
- First, enable PostGIS extension in Supabase:
  CREATE EXTENSION IF NOT EXISTS postgis;
  CREATE EXTENSION IF NOT EXISTS postgis_topology;
- Create the PARCEL table migration:
  CREATE TABLE parcel (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v7(),
    town_id UUID NOT NULL REFERENCES town(id),
    state_id TEXT,                                    -- GeoLibrary STATE_ID (primary join key)
    map_bk_lot TEXT,                                  -- Map/Book/Lot (what boards reference verbally)
    geometry GEOMETRY(Polygon, 4326),                 -- PostGIS polygon in WGS 84
    prop_loc TEXT,                                    -- Property location (street address from parcel data)
    town_name TEXT,                                   -- Town name from GeoLibrary (uppercase)
    county TEXT,
    geocode TEXT,                                     -- 5-char town geocode
    owner1 TEXT,                                      -- Primary owner
    owner2 TEXT,                                      -- Secondary owner
    own_addr1 TEXT,                                   -- Owner mailing address line 1
    own_addr2 TEXT,                                   -- Owner mailing address line 2
    own_city TEXT,
    own_state TEXT,
    own_zip TEXT,
    land_val DECIMAL(12,2),                           -- Assessed land value
    bldg_val DECIMAL(12,2),                           -- Assessed building value
    land_use TEXT,                                    -- Land use code
    land_use_desc TEXT,                               -- Land use description
    lot_size DECIMAL(10,2),                           -- Lot size
    lot_units TEXT,                                   -- Lot size units (A=acres)
    bldg_size DECIMAL(10,2),                          -- Building size (sq ft)
    ls_date DATE,                                     -- Last sale date
    ls_price DECIMAL(12,2),                           -- Last sale price
    ls_book TEXT,                                     -- Deed book reference
    ls_page TEXT,                                     -- Deed page reference
    shape_area DOUBLE PRECISION,                      -- Area in source units
    source TEXT NOT NULL DEFAULT 'geolibrary',        -- geolibrary or manual
    fmupdat DATE,                                     -- Last update date from GeoLibrary
    imported_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE(town_id, state_id)
  );
  CREATE INDEX idx_parcel_town ON parcel(town_id);
  CREATE INDEX idx_parcel_map_bk_lot ON parcel(town_id, map_bk_lot);
  CREATE INDEX idx_parcel_geometry ON parcel USING GIST(geometry);
  CREATE INDEX idx_parcel_geocode ON parcel(geocode);
  CREATE INDEX idx_parcel_owner ON parcel(town_id, owner1);
- The GIST index on geometry enables fast spatial queries (ST_DWithin, ST_Intersects)
- RLS policies: admin/staff can read all parcels for their town; public can read (parcels are public records)

TASK 4: Create parcel import job
- Create packages/api/src/services/parcels/import-parcels.ts
- Query the Maine GeoLibrary ArcGIS REST API for a specific town:
  GET https://services1.arcgis.com/RbMX0mRVOFNTdLzd/arcgis/rest/services/Maine_Parcels_Merged/FeatureServer/1/query
    ?where=TOWN='NEWCASTLE'
    &outFields=*
    &f=geojson
    &resultOffset=0
    &resultRecordCount=2000
- Handle pagination:
  - The API returns max 2000 records per request
  - Check the response for exceededTransferLimit: true
  - If exceeded, increment resultOffset by 2000 and make another request
  - Continue until all records are retrieved
  - Log progress: "Importing parcels for Newcastle: page 1 (2000 records), page 2 (1847 records), done (3847 total)"
- For each GeoJSON feature in the response:
  1. Extract properties and map to PARCEL table columns:
     - STATE_ID -> state_id
     - MAP_BK_LOT -> map_bk_lot
     - PROP_LOC -> prop_loc
     - TOWN -> town_name
     - COUNTY -> county
     - GEOCODE -> geocode
     - OWNER1 -> owner1 (from Merged service inline data)
     - LAND_VAL -> land_val
     - ... (map all 51 fields from the Merged service)
  2. Extract geometry (GeoJSON polygon) and convert to PostGIS format:
     - Use ST_GeomFromGeoJSON(geometry_json) to store the polygon
     - The geometry is already in WGS 84 (EPSG:4326) because we requested f=geojson
  3. Upsert into PARCEL table (ON CONFLICT (town_id, state_id) DO UPDATE)
- Use uppercase town name matching (GeoLibrary uses uppercase: 'NEWCASTLE', not 'Newcastle')
- Store the town's GEOCODE from the first parcel record for future queries (Task 7)
- Transaction: process the import within a database transaction — rollback on failure

TASK 5: Create E911 address import job
- Create packages/api/src/services/parcels/import-addresses.ts
- Same pagination approach as parcels:
  GET https://services1.arcgis.com/RbMX0mRVOFNTdLzd/arcgis/rest/services/Addresses/FeatureServer/0/query
    ?where=TOWN='NEWCASTLE'
    &outFields=*
    &f=geojson
    &resultOffset=0
    &resultRecordCount=2000
- For each GeoJSON feature:
  1. Map properties to E911_ADDRESS columns
  2. Extract the point geometry (latitude, longitude from properties; PostGIS point from GeoJSON geometry)
  3. Create the PostGIS point: ST_SetSRID(ST_MakePoint(longitude, latitude), 4326)
  4. Upsert into E911_ADDRESS table (ON CONFLICT (town_id, site_uid) DO UPDATE)
- E911 addresses link to parcels via STATE_ID and MAP_BK_LOT — these links enable the combined data model from advisory 2.3

TASK 6: Handle coordinate system
- Per advisory 2.3: requesting f=geojson from the ArcGIS REST API automatically reprojects coordinates to WGS 84 (EPSG:4326)
- This is the standard coordinate system for web mapping (Leaflet, Mapbox GL)
- Our PostGIS tables use SRID 4326 to match
- Verification step at import time:
  1. After importing the first batch of parcels, query one parcel's geometry:
     SELECT ST_AsGeoJSON(geometry) FROM parcel WHERE town_id = :townId LIMIT 1;
  2. Verify the coordinates are in the expected range for Maine:
     - Longitude: approximately -66.9 to -71.1
     - Latitude: approximately 43.0 to 47.5
  3. If coordinates are outside this range, log a warning: "Coordinate system may not be WGS 84 — manual verification required"
  4. If coordinates appear to be in the native projection (WKID 26919 — large numbers in meters), fall back to using outSR=4326 in the API request to force reprojection
- Create a utility function validateCoordinates(lng, lat): boolean that checks Maine bounding box

TASK 7: Store town GEOCODE
- The GEOCODE field is a 5-character string that uniquely identifies each municipality in GeoLibrary
- Store it on the TOWN record:
  ALTER TABLE town ADD COLUMN geocode TEXT;
- During the first import for a town, extract the GEOCODE from the parcel data and store it:
  UPDATE town SET geocode = :geocode WHERE id = :townId;
- For subsequent imports, use the GEOCODE for more reliable matching:
  - Instead of where=TOWN='NEWCASTLE', use where=GEOCODE='06340' (example)
  - This avoids issues with town name variations (e.g., 'EAST MACHIAS' vs 'EAST_MACHIAS')
- Create a lookup endpoint: GET /api/geolibrary/towns to query the GeoLibrary town boundaries service for available towns and their geocodes:
  GET https://services1.arcgis.com/RbMX0mRVOFNTdLzd/arcgis/rest/services/Maine_Town_and_Townships_Boundary_Polygons/FeatureServer/0/query
    ?where=1=1
    &outFields=TOWN,COUNTY,GEOCODE
    &returnDistinctValues=true
    &f=json
- This lets the admin select their town from the GeoLibrary list during setup

TASK 8: Create automated monthly check
- Create packages/api/src/services/parcels/refresh-checker.ts
- The monthly check determines if GeoLibrary has new data for each configured town:
  1. Query the API for the town's current record count:
     GET .../FeatureServer/1/query?where=TOWN='NEWCASTLE'&returnCountOnly=true&f=json
  2. Compare against the stored count from the last import (PARCEL_IMPORT_LOG table)
  3. If the count differs (or if it has been >30 days since last check): trigger a full import
  4. After import, send a notification to the admin: "Parcel data updated for Newcastle: 3,847 parcels imported (12 new, 5 updated)"
- Create PARCEL_IMPORT_LOG table:
  id UUID PRIMARY KEY,
  town_id UUID NOT NULL REFERENCES town(id),
  data_type TEXT NOT NULL,                           -- 'parcels' or 'e911'
  records_imported INTEGER,
  records_updated INTEGER,
  records_deleted INTEGER,
  previous_count INTEGER,
  new_count INTEGER,
  status TEXT NOT NULL,                              -- 'completed', 'failed', 'in_progress'
  error_message TEXT,
  started_at TIMESTAMPTZ NOT NULL,
  completed_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
- Schedule the monthly check:
  - Create a Fastify route POST /api/parcels/check-updates (admin-only) for manual trigger
  - Create a scheduled job (using node-cron or similar): runs on the 1st of each month at 3 AM ET
  - The job iterates through all towns with geocode configured and checks each one
- The check should be lightweight — only the record count query, not a full data download
