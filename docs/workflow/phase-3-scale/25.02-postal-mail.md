# Session 25.02 — Postal Mail Notifications

**Phase:** 3 — Scale | **Block:** 25 — Proximity Matching & Notifications
**Dependencies:** 25.01
**Estimated tasks:** 7

---

## Description

Build the postal mail notification system for required land-use notices. Many municipalities are legally required to send physical notice letters to abutting property owners before certain hearings (planning, zoning, conservation). This session creates the letter generation pipeline using Puppeteer, mailing label generation, postal mail tracking for compliance records, and an admin dashboard for managing the postal notice workflow.

## Tasks

1. Create postal notice generation: for required land-use notifications, generate physical letters to abutters
2. Use abutters list from proximity matching (25.01) to identify recipients
3. Generate notice letters via Puppeteer: formal letter format with parcel info, hearing date/time, board name, town seal
4. Create mailing labels generation: PDF with address labels for all recipients (from parcel owner addresses in ADB data)
5. Create postal mail tracking: record that notices were generated, who received them, date generated
6. Create admin postal mail dashboard: generate notices, print labels, mark as mailed
7. Verify: notice letters generate with correct recipient info, labels print correctly, tracking records created

## Prompt

```
You are building the postal mail notification system for the Town Meeting Manager — a municipal meeting management platform for Maine towns. Many land-use boards are legally required to send certified postal mail notices to abutting property owners before certain hearings (subdivision reviews, site plan approvals, variances, conditional use permits). This system generates the formal notice letters and mailing labels from the abutters list data.

This feature builds on:
- Parcel data and abutters lists from sessions 24.01-24.02 (PostGIS spatial queries for nearby parcels)
- Proximity matching from session 25.01 (identifying affected properties and residents)
- PDF generation via Puppeteer (same approach as agenda packets from session 06.03 and warrant documents from session 18.01)

PROJECT CONTEXT:
- Monorepo: packages/api (Fastify backend), packages/web (React 19), packages/shared (types, schemas)
- PostGIS-enabled PostgreSQL with PARCEL table containing owner names and mailing addresses
- Puppeteer for PDF generation (installed and configured in session 06.03)
- Tailwind CSS v4 + shadcn/ui for components
- The system generates PDFs for printing — it does NOT send physical mail electronically (no Lob/direct mail API integration)
- Town staff print the letters and labels, then mail them manually via USPS

TASK 1: Create postal notice generation service
- Create packages/api/src/services/postal/postal-notice-service.ts
- The service generates formal notice letters for a specific agenda item's abutters:
  interface PostalNoticeRequest {
    agendaItemId: string;
    meetingId: string;
    boardId: string;
    townId: string;
    parcelId: string;          // the subject parcel
    radius?: number;           // override default radius (feet)
    noticeType: 'public_hearing' | 'abutter_notice' | 'site_plan' | 'subdivision' | 'variance' | 'custom';
    customText?: string;       // additional notice text
    generatedBy: string;       // person_id of the admin generating notices
  }
  interface PostalNoticeResult {
    lettersPdfUrl: string;     // URL to the generated letters PDF
    labelsPdfUrl: string;      // URL to the generated mailing labels PDF
    recipientCount: number;
    trackingId: string;        // reference for the postal tracking record
  }
- The service:
  1. Queries the abutters list using the spatial query from session 24.02 Task 6
  2. For each abutter, extracts the owner mailing address from the PARCEL table (OWN_ADDR1, OWN_ADDR2, OWN_CITY, OWN_STATE, OWN_ZIP)
  3. Generates individual notice letters (one per owner — merged into a single PDF)
  4. Generates a mailing labels sheet (Avery 5160 format — standard address labels)
  5. Stores the PDFs in Supabase Storage
  6. Creates tracking records

TASK 2: Use abutters list for recipients
- The recipient list comes from the abutters query (session 24.02):
  SELECT p.id, p.map_bk_lot, p.owner1, p.owner2,
         p.own_addr1, p.own_addr2, p.own_city, p.own_state, p.own_zip,
         p.prop_loc,
         ST_Distance(p.geometry::geography, subject.geometry::geography) AS distance_meters
  FROM parcel p
  CROSS JOIN (SELECT geometry FROM parcel WHERE id = :subjectParcelId) AS subject
  WHERE p.town_id = :townId
    AND p.id != :subjectParcelId
    AND ST_DWithin(p.geometry::geography, subject.geometry::geography, :radiusMeters)
  ORDER BY distance_meters ASC;
- Filter out parcels without owner mailing addresses (own_addr1 is null) — log a warning for each
- De-duplicate by owner mailing address: if the same owner has multiple abutting parcels, send only one letter listing all their affected parcels
- Include the subject parcel's owner as a recipient (the applicant/owner of the property under review)
- Create a recipient list interface:
  interface PostalRecipient {
    parcelId: string;
    mapBkLot: string;
    ownerName: string;         // OWNER1 (+ OWNER2 if present)
    mailingAddress: {
      line1: string;
      line2?: string;
      city: string;
      state: string;
      zip: string;
    };
    propertyAddress: string;   // the abutter's property address
    distanceFeet: number;      // distance from the subject parcel
  }

TASK 3: Generate notice letters via Puppeteer
- Create packages/api/src/services/postal/letter-generator.ts
- Create an API endpoint: POST /api/postal/generate-notices
- Generate a formal notice letter PDF using Puppeteer:
  - Letter format: standard 8.5" x 11" with 1" margins
  - Letter content:
    - Header: Town name, town seal (if uploaded), board name
    - Date: notice generation date
    - Recipient address block: owner name and mailing address
    - Salutation: "Dear Property Owner:" (generic — applicable to all abutters)
    - Notice body (formal language):
      "You are hereby notified that the [Board Name] of the Town of [Town] will hold a [hearing type] on [date] at [time] at [location] regarding the following application:"
    - Application/agenda item details:
      - "Applicant: [name, if available]"
      - "Property: [subject parcel address] (Map [X], Lot [Y])"
      - "Proposal: [agenda item title and description]"
      - "Your property at [abutter's address] (Map [X], Lot [Y]) is within [distance] feet of the subject property."
    - Public participation notice:
      "You are invited to attend the hearing and provide comment. Written comments may also be submitted to [town contact] prior to the hearing."
    - Contact information: town office address, phone, email
    - Closing: board name, town, date
    - Footer: "This notice is provided in accordance with [applicable statute reference]"
  - Each recipient gets their own letter (personalized with their name, address, and property reference)
  - All letters are merged into a single PDF for easy printing (page break between letters)
- The generated PDF is stored in Supabase Storage: {town_id}/postal-notices/{meeting_id}/{agenda_item_id}/notices.pdf
- Return the storage URL for download

TASK 4: Create mailing labels generation
- Create packages/api/src/services/postal/label-generator.ts
- Generate a PDF formatted for standard mailing labels:
  - Format: Avery 5160 (or compatible) — 30 labels per sheet, 3 columns x 10 rows
  - Label size: 2.625" x 1" (66.7mm x 25.4mm)
  - Each label contains:
    - Owner name (OWNER1, and OWNER2 on second line if present)
    - Mailing address (OWN_ADDR1, OWN_ADDR2)
    - City, State ZIP (OWN_CITY, OWN_STATE OWN_ZIP)
  - Font: 10pt standard font (Arial or similar) for readability
  - Labels are ordered by ZIP code for efficient USPS processing
- Also generate a return address label option:
  - Town office name and address as the sender
  - These appear on a separate label page
- The labels PDF is stored alongside the letters: {town_id}/postal-notices/{meeting_id}/{agenda_item_id}/labels.pdf
- Handle edge cases:
  - Very long owner names: truncate at 2 lines
  - Missing address components: omit blank fields
  - PO Box addresses: format appropriately
  - Out-of-state owners: include full state name or abbreviation

TASK 5: Create postal mail tracking
- Create POSTAL_NOTICE_BATCH table:
  CREATE TABLE postal_notice_batch (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v7(),
    town_id UUID NOT NULL REFERENCES town(id),
    meeting_id UUID NOT NULL REFERENCES meeting(id),
    agenda_item_id UUID NOT NULL REFERENCES agenda_item(id),
    subject_parcel_id UUID NOT NULL REFERENCES parcel(id),
    notice_type TEXT NOT NULL,
    radius_feet DECIMAL(8,2) NOT NULL,
    recipient_count INTEGER NOT NULL,
    letters_pdf_path TEXT,                            -- Supabase Storage path
    labels_pdf_path TEXT,                             -- Supabase Storage path
    status TEXT NOT NULL DEFAULT 'generated',         -- generated, printed, mailed, confirmed
    generated_by UUID NOT NULL REFERENCES person(id),
    generated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    printed_at TIMESTAMPTZ,
    mailed_at TIMESTAMPTZ,
    mailed_by UUID REFERENCES person(id),
    mail_type TEXT,                                    -- first_class, certified, certified_return_receipt
    tracking_numbers TEXT[],                           -- USPS tracking numbers if certified
    notes TEXT,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
  );
- Create POSTAL_NOTICE_RECIPIENT table:
  CREATE TABLE postal_notice_recipient (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v7(),
    batch_id UUID NOT NULL REFERENCES postal_notice_batch(id) ON DELETE CASCADE,
    parcel_id UUID NOT NULL REFERENCES parcel(id),
    owner_name TEXT NOT NULL,
    mailing_address JSONB NOT NULL,                   -- structured address
    property_address TEXT,
    distance_feet DECIMAL(8,2),
    status TEXT NOT NULL DEFAULT 'generated',          -- generated, mailed, returned, undeliverable
    tracking_number TEXT,                              -- individual tracking if certified
    return_reason TEXT,                                -- if mail returned: insufficient_address, vacant, refused, etc.
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
  );
- This tracking creates a compliance record proving that notices were sent as required by statute
- The batch tracks the overall mailing; individual recipients track per-letter status

TASK 6: Create admin postal mail dashboard
```
