# Session 24.02 — Parcel Display & Agenda Item Linking

**Phase:** 3 — Scale | **Block:** 24 — Parcel Data Integration
**Dependencies:** 24.01
**Estimated tasks:** 9

---

## Description

Build the parcel map display, search, and linking system that connects imported parcel data to the meeting workflow. This session creates an interactive map component showing parcel boundaries and E911 address points, a parcel search interface, parcel detail panels, the ability to link parcels to agenda items for land-use boards, abutters list generation using PostGIS spatial queries, and a public portal parcel display.

## Tasks

1. Create parcel map component using Leaflet or Mapbox GL JS: display parcel polygons and E911 points on a map
2. Create parcel search: search by MAP_BK_LOT, address, or owner name -> display on map
3. Create parcel detail panel: show parcel info (owner, address, assessed values, land use, lot size, last sale)
4. Link parcels to agenda items: when creating an agenda item for a land-use board, search and attach relevant parcels
5. Auto-populate agenda item details from parcel data when a parcel is linked
6. Create abutters list generation: spatial query for parcels within a configurable radius of the linked parcel (PostGIS ST_DWithin)
7. Display linked parcels on public portal agenda view
8. Create parcel lookup widget for meeting operator: quick search during live meeting
9. Verify: map displays parcels, search works, linking to agenda items works, abutters list generates correctly

TASK 2: Create parcel search
- Create packages/web/src/components/parcels/ParcelSearch.tsx
- Search input with three modes:
  1. MAP_BK_LOT search: type "003-059" or "Map 3 Lot 59" — find parcel by lot reference
  2. Address search: type "123 Main" — find E911 addresses matching, then show the associated parcel
  3. Owner name search: type "Smith" — find parcels with OWNER1 or OWNER2 matching
- Create an API endpoint: GET /api/towns/:townId/parcels/search?q=QUERY&type=lot|address|owner
  - For lot search: SELECT * FROM parcel WHERE town_id = :townId AND map_bk_lot ILIKE '%' || :q || '%'
  - For address search: SELECT e.*, p.map_bk_lot FROM e911_address e LEFT JOIN parcel p ON e.state_id = p.state_id WHERE e.town_id = :townId AND e.address ILIKE '%' || :q || '%'
  - For owner search: SELECT * FROM parcel WHERE town_id = :townId AND (owner1 ILIKE '%' || :q || '%' OR owner2 ILIKE '%' || :q || '%')
  - Limit results to 20, ordered by relevance (exact match first, then partial)
- Search results dropdown (shadcn/ui Command/Combobox pattern):

TASK 6: Create abutters list generation
- Create an API endpoint: GET /api/parcels/:parcelId/abutters?radius=500&unit=feet
- Use PostGIS spatial query to find nearby parcels:
  SELECT p.*, e.address
  FROM parcel p
  LEFT JOIN e911_address e ON p.state_id = e.state_id AND p.town_id = e.town_id
  WHERE p.town_id = :townId
    AND p.id != :parcelId
    AND ST_DWithin(
      p.geometry::geography,
      (SELECT geometry::geography FROM parcel WHERE id = :parcelId),
      :radiusMeters
    )
  ORDER BY ST_Distance(
    p.geometry::geography,
    (SELECT geometry::geography FROM parcel WHERE id = :parcelId)
  );
- Note: ST_DWithin with ::geography casting uses meters; convert feet to meters in the API (1 foot = 0.3048 meters)
- Default radius: 500 feet (configurable per board in admin settings)
- The abutters list includes:
  - Parcel MAP_BK_LOT
  - Owner name(s)
  - Owner mailing address (for postal notices — session 25.02)
  - Property address
  - Distance from the subject parcel
- Create packages/web/src/components/parcels/AbuttersList.tsx:
  - Table display of abutters with all fields
  - Map view: highlight the subject parcel (blue) and abutters (orange) on the map
  - Export options: CSV, PDF (for mailing labels — feeds into session 25.02)
  - Count display: "23 abutting properties within 500 feet"
- The abutters list is used for:
  - Required land-use notifications (mailing certified letters to property owners near a proposed project)
  - Proximity-based notifications to registered residents (session 25.01)
