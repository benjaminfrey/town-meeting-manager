# Session 28.01 — Tests: All Phase 3 Features

**Phase:** 3 — Scale | **Block:** 28 — Testing
**Dependencies:** 24.02, 25.02, 26.01, 27.01
**Estimated tasks:** 10

---

## Description

Write comprehensive tests for all Phase 3 features: GeoLibrary parcel import pipeline, parcel map display and agenda item linking, proximity matching with resident notifications, postal mail generation, multi-town consortium management, and advanced zoning/planning features. This session validates the entire Phase 3 feature set works correctly end-to-end.

## Tasks

1. Write integration tests for GeoLibrary import: ArcGIS REST API query construction, pagination handling (2000-record batches via resultOffset), PostGIS geometry storage, coordinate system handling (GeoJSON auto-reprojection to WGS 84)
2. Write component tests for parcel map display: Leaflet/Mapbox rendering, parcel polygon display, E911 point markers, search by MAP_BK_LOT/address/owner
3. Write component tests for parcel detail panel: owner info, assessed values, land use, lot size from combined parcel + ADB + E911 data
4. Write integration tests for parcel-agenda item linking: search parcels, attach to agenda item, auto-populate item details from parcel data
5. Write integration tests for abutters list generation: PostGIS ST_DWithin spatial query with configurable radius, correct neighbor identification
6. Write integration tests for proximity matching: resident address geocoding against E911 data, spatial query for nearby parcels, notification dispatch to affected residents via email and SMS
7. Write integration tests for postal mail generation: notice letter content accuracy (parcel info, hearing date, board name, town seal), mailing label PDF generation, tracking record creation
8. Write component tests for multi-town consortium: create consortium, add/remove towns, cross-town dashboard aggregation, town switcher, shared staff accounts across towns
9. Write component tests for zoning/planning features: application creation and workflow (submitted → review → hearing → approved/denied), conditions of approval tracking, zoning district display
10. Write E2E tests for complete Phase 3 workflows: (a) import parcels → link to agenda item → generate abutters list → proximity notification → postal notice, (b) multi-town setup → cross-town dashboard → town switching, (c) zoning application → hearing → decision with conditions
