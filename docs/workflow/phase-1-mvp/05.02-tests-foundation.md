# Session 05.02 — Tests: Shared Package, Auth, Onboarding

**Phase:** 1 — MVP | **Block:** 05 — Testing -- Foundation
**Dependencies:** 05.01, 03.03, 04.03
**Estimated tasks:** 12

---

## Description

Write the foundational test suite covering the shared package (Zod schemas, utility functions, permission logic), authentication UI, the onboarding wizard, and the member management system. This includes unit tests for business logic, component tests for form validation and UI interactions, integration tests verifying database writes through the wizard flow, and E2E tests for the complete onboarding and member management paths.

## Tasks

1. Write unit tests for shared package Zod schemas: valid/invalid data for Person, UserAccount, Board, Meeting, AgendaItem, Motion
2. Write unit tests for shared utility functions: generateId, formatGovernmentTitle, calculateQuorum
3. Write unit tests for permission constants and permission checking logic
4. Write component tests for LoginPage: form validation, successful login flow, error display
5. Write component tests for AuthProvider: auth state management, token refresh handling
6. Write component tests for WizardStage1-5: field validation, conditional rendering, navigation
7. Write integration test for wizard completion: all 5 stages -> submit -> verify database records created (TOWN, BOARD, PERSON, USER_ACCOUNT)
8. Write component tests for PermissionMatrixEditor: template application, per-board overrides, mutual exclusivity enforcement
9. Write component tests for MemberRoster: add member, archive member, role transition
10. Write E2E test: complete onboarding flow (login -> wizard -> dashboard)
11. Write E2E test: add a board member with permissions
12. Run full test suite and verify all tests pass

TASK 3: Unit tests for permission logic
- File: packages/shared/src/permissions/__tests__/permissions.test.ts
- hasPermission(permissions, action, boardId):
  - Returns true for action in global permissions: { global: { A1: true } } -> hasPermission(p, 'A1', 'any-board') -> true
  - Returns false for denied action: { global: { A1: false } } -> hasPermission(p, 'A1', 'any-board') -> false
  - Board override takes precedence: { global: { M1: false }, board_overrides: { 'board-1': { M1: true } } } -> hasPermission(p, 'M1', 'board-1') -> true
  - Board override does not affect other boards: same permissions, hasPermission(p, 'M1', 'board-2') -> false
  - Missing action defaults to false
- checkRoleMutualExclusivity(person, targetRole):
  - Staff person + target board_member -> returns conflict error with message
  - Board_member person + target staff -> returns conflict error with message
  - Admin person + target board_member -> returns no conflict (allowed)
  - Person with no user_account + any target -> returns no conflict
  - Staff person + target staff -> returns no conflict (same role)
- getTemplatePermissions(templateName):
  - 'town_clerk' template includes A1-A6, M1-M7, R1-R6, C1-C5 all set to true
  - 'general_staff' template includes only A3, R4, R6 set to true
  - 'recording_secretary_only' includes M2-M5, R1-R4, R6 (board-specific)
  - All templates exclude T1-T4 (admin-only actions)
  - V1-V5 are always true for all templates
