# Session 01.06 — Database Schema: Minutes, Templates, Exhibits, Notifications

**Phase:** 1 — MVP | **Block:** 01 — Infrastructure & Monorepo Foundation
**Dependencies:** 01.05
**Estimated tasks:** 13

---

## Description

Extend the database schema with tables for minutes documents, agenda templates, exhibits/attachments, notification events and deliveries, subscriber preferences, town notification configuration, permission templates, and an audit log. These tables complete the data model required for the full MVP feature set.

## Tasks

1. Create migration: MINUTES_DOCUMENT table (id UUID PK, meeting_id FK UNIQUE, town_id FK, status enum, content_json JSONB, html_rendered TEXT, pdf_storage_path, generated_by enum, approved_at, approved_by_motion_id FK, created_at, updated_at)
2. Create migration: MINUTES_SECTION table (id UUID PK, minutes_document_id FK, town_id FK, section_type, sort_order, title, content_json JSONB, source_agenda_item_id FK, created_at, updated_at)
3. Create migration: AGENDA_TEMPLATE table (id UUID PK, board_id FK, town_id FK, name, is_default, sections JSONB, created_at, updated_at)
4. Create migration: EXHIBIT table (id UUID PK, agenda_item_id FK, town_id FK, title, file_storage_path, file_type, file_size, exhibit_type, uploaded_by FK, visibility enum, sort_order, created_at)
5. Create migration: NOTIFICATION_EVENT table (id UUID PK, town_id FK, event_type, payload JSONB, status enum, created_at, processed_at)
6. Create migration: NOTIFICATION_DELIVERY table (id UUID PK, event_id FK, town_id FK, subscriber_id FK, channel enum, status enum, external_id, error_message, created_at, delivered_at)
7. Create migration: SUBSCRIBER_NOTIFICATION_PREFERENCE table (id UUID PK, person_id FK, town_id FK, channel enum, event_type, enabled boolean, consent_timestamp, consent_method, consent_record)
8. Create migration: TOWN_NOTIFICATION_CONFIG table (id UUID PK, town_id FK UNIQUE, postmark_server_token_encrypted, postmark_sender_email, postmark_sender_name, twilio config fields)
9. Create migration: PERMISSION_TEMPLATE table (id UUID PK, town_id FK, name, description, permissions JSONB, is_system_default, created_at)
10. Create migration: AUDIT_LOG table (id UUID PK, town_id FK, user_account_id FK, action, entity_type, entity_id, details JSONB, created_at)
11. Seed default permission templates (Town Clerk, Deputy Clerk, Board-Specific Staff, General Staff, Recording Secretary Only)
12. Add indexes on notification tables for efficient delivery tracking queries
13. Run migrations and verify all tables created correctly

KEY CONTEXT:

MINUTES ARCHITECTURE (from advisory 2.2 and 4.3):
- Minutes are stored as structured JSON (canonical source)
- Rendered to HTML for the public portal
- PDF generated on demand via Puppeteer
- Status lifecycle: draft → review → approved (by board vote) → published
- Minutes are per-meeting (one minutes_document per meeting)
- Minutes sections correspond to agenda items

NOTIFICATION ARCHITECTURE (from advisory 2.4):
- Unified notification event system serving email (Postmark, Phase 1) and SMS (Twilio, Phase 2)
- Event triggers create notification_event records
- System queries subscribers by event_type + channel preference
- Deliveries are tracked individually per subscriber per channel
- TCPA compliance for SMS: consent tracking with timestamp, method, and exact text shown

PERMISSION TEMPLATES (from advisory 1.2):
- 5 default templates that serve as starting points for new staff accounts:
  1. Town Clerk — full operational access (A1-A7 minus A4/A7, M1-M7, R1-R6, C1-C5)
  2. Deputy Clerk — minutes and records focused (A2,A3,A6, M1-M5, R1-R4,R6)
  3. Board-Specific Staff — full access on designated boards only
  4. General Staff — view-oriented (A3, R4,R6)
  5. Recording Secretary Only — meeting recording and minutes only (M2-M5, R1-R4,R6)

TASK 1: Create MINUTES_DOCUMENT table (20260308000014)
```sql
CREATE TABLE minutes_document (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  meeting_id UUID NOT NULL UNIQUE REFERENCES meeting(id) ON DELETE CASCADE,
  town_id UUID NOT NULL REFERENCES town(id) ON DELETE CASCADE,
  status minutes_document_status NOT NULL DEFAULT 'draft',
  content_json JSONB NOT NULL DEFAULT '{}',
  html_rendered TEXT,
  pdf_storage_path TEXT,
  generated_by minutes_generated_by NOT NULL DEFAULT 'manual',
  approved_at TIMESTAMPTZ,
  approved_by_motion_id UUID REFERENCES motion(id),
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

COMMENT ON TABLE minutes_document IS 'The minutes for a meeting. One per meeting. JSON is the canonical source; HTML and PDF are rendered outputs.';
COMMENT ON COLUMN minutes_document.content_json IS 'Structured JSON representation of the complete minutes. Source of truth for rendering.';
COMMENT ON COLUMN minutes_document.html_rendered IS 'Pre-rendered HTML for the public portal. Regenerated when content_json changes.';
COMMENT ON COLUMN minutes_document.pdf_storage_path IS 'Path in Supabase Storage to the generated PDF.';
COMMENT ON COLUMN minutes_document.approved_by_motion_id IS 'The motion by which the board voted to approve these minutes.';
```
