# Session 04.01 — Town Profile Dashboard

**Phase:** 1 — MVP | **Block:** 04 — Town Profile & Board Management
**Dependencies:** 03.03
**Estimated tasks:** 10

---

## Description

Build the Town Profile Dashboard — the home screen users land on after completing the onboarding wizard. This page shows a progress checklist tracking setup completion, collapsible sections displaying configured town settings (one per wizard stage), and inline editors for modifying each setting. All data flows through TanStack Query with Supabase so the dashboard updates instantly when settings change.

## Tasks

1. Create TownDashboard page at /dashboard route using clientLoader with queryClient.ensureQueryData() to load town data
2. Create progress checklist component (top of dashboard): shows setup completion status with items from wizard completion spec (add board members, set meeting notice template, configure minutes approval, upload town seal, set portal subdomain, acknowledge retention policy)
3. Create collapsible settings sections (one per wizard stage) showing configured values with Edit buttons
4. Create TownSettingsEditor for inline editing of town profile fields (town name, type, population, contact info)
5. Create MeetingDefaultsEditor for editing meeting formality and minutes style defaults
6. Implement presiding officer and minutes recorder default editors
7. Create town seal upload component (Supabase Storage integration)
8. Create retention policy acknowledgment modal with policy text and acknowledge button
9. Use TanStack Query useQuery hooks for reactive data with cache invalidation on mutations
10. Verify: dashboard shows wizard-configured data, edit forms save correctly, progress checklist tracks completion

Read docs/advisory-resolutions/2.1-onboarding-wizard-ux-spec.md for the post-wizard dashboard specification, then implement the following.

TASK 1: Create the TownDashboard page component
- Route: /dashboard (this is the app's home screen after login)
- File: packages/web/src/routes/dashboard.tsx
- Use clientLoader with queryClient.ensureQueryData(queryKeys.town(townId), () => fetchTown(townId)) to load the TOWN record
- Also load board count, member count, and checklist completion status in the clientLoader
- Export the route component using React Router v7 Framework Mode conventions

TASK 2: Create the ProgressChecklist component
- File: packages/web/src/components/dashboard/ProgressChecklist.tsx
- Render as a Card at the top of the dashboard with a heading like "Complete Your Setup"
- Checklist items (from advisory 2.1 post-wizard specification):
  1. "Add board members (0 of N added)" — links to /boards, required before first meeting
  2. "Set meeting notice template" — links to town profile settings section
  3. "Configure minutes approval workflow" — links to town profile settings section
  4. "Upload town seal" — links to town profile settings section
  5. "Set public portal subdomain" — links to town profile settings section
  6. "Acknowledge retention policy" — opens modal, required before first meeting
- Each item shows a checkbox (checked when complete), description, and link/button

- The checklist hides entirely when all items are complete — replaced with a "Setup complete" success message
- Use TanStack Query useQuery hooks to reactively track completion status; invalidate on relevant mutations

TASK 3: Create collapsible settings sections
- File: packages/web/src/components/dashboard/SettingsSection.tsx
- One Accordion section per wizard stage:
  - "Your Town" — town name, state, municipality type, population, contact info
  - "Governing Board" — board name, member count, election method, officer election method
  - "Meeting Roles" — presiding officer default, minutes recorder default, staff roles present
  - "Boards & Committees" — list of all boards with member counts
  - "Meeting Style & Minutes" — formality setting, minutes style setting

- Use shadcn/ui Accordion component for collapsible behavior

TASK 4: Create TownSettingsEditor component
- File: packages/web/src/components/dashboard/TownSettingsEditor.tsx
- Inline editor (replaces read-only view when Edit is clicked) for:
  - Town name (text input, 2-100 chars, allows hyphens/apostrophes/periods)
  - State (dropdown: Maine, New Hampshire, Vermont, Massachusetts, Connecticut, Rhode Island)
  - Municipality type (dropdown: Town, City, Plantation)
  - Population range (dropdown: Under 1,000 / 1,000-2,500 / 2,500-5,000 / 5,000-10,000 / Over 10,000)
  - Primary contact name (text input, 2-100 chars)
  - Primary contact role (text input, max 100 chars)
- Use React Hook Form + Zod schema from @town-meeting/shared for validation
- On save: call useUpdateTown() mutation → supabase.from('town').update() → queryClient.invalidateQueries(queryKeys.town(townId))
- Show Save and Cancel buttons; Cancel reverts to read-only view

TASK 5: Create MeetingDefaultsEditor component
- File: packages/web/src/components/dashboard/MeetingDefaultsEditor.tsx
- Edit meeting formality (radio buttons with descriptions):
  - "Open (informal)" — Discussion flows freely. Most small Maine towns operate this way.
  - "Structured (semi-formal)" — Chair follows agenda in order. Members recognized before speaking.
  - "Formal (Robert's Rules)" — Strict parliamentary procedure.
- Edit minutes style (radio buttons with descriptions):
  - "Action minutes" — Records only decisions, motions, votes.
  - "Summary minutes" — Decisions plus brief discussion summaries. Most common.
  - "Narrative minutes" — Detailed account including who said what.
- These are town-level defaults that apply to all boards unless overridden per board
- Use React Hook Form with Zod validation
- Save calls useUpdateTown() mutation to write to Supabase, then invalidates the town query

TASK 6: Create presiding officer and minutes recorder editors
- Presiding officer dropdown: Chair of the board, Town Manager, Town Administrator, Mayor, Moderator
- Minutes recorder dropdown: Town Clerk, Deputy Clerk, Recording Secretary (board member), Recording Secretary (staff), Other staff
- These set defaults for new meetings; they can be overridden per meeting
- Same inline edit pattern as other editors

TASK 7: Create TownSealUpload component
- File: packages/web/src/components/dashboard/TownSealUpload.tsx
- Displays current town seal image if uploaded, or a placeholder
- Upload button opens file picker (accept: image/png, image/jpeg, image/svg+xml)
- On file select: upload to Supabase Storage bucket "town-seals" under path {town_id}/seal.{ext}
- Show upload progress indicator
- On success: update TOWN record with seal_url, checklist item marks as complete
- Image preview with a "Remove" option
- Max file size: 2 MB with client-side validation

TASK 8: Create RetentionPolicyModal component
- File: packages/web/src/components/dashboard/RetentionPolicyModal.tsx
- Use shadcn/ui Dialog component
- Display retention policy text explaining:
  - Name and government title retained indefinitely (public record)
  - Vote records, motions, attendance retained indefinitely (public record)
  - Email/phone scrubbed after configurable period post-departure (default 1 year)
  - Login credentials deleted immediately upon archival
  - Per advisory 1.2 data retention rules
- "I acknowledge this retention policy" button at the bottom
- On acknowledge: write acknowledgment timestamp to TOWN record, close modal
- Cannot be dismissed without acknowledging (no close-on-outside-click)

TASK 9: Wire up TanStack Query reactive data
- Every editor and display component uses useQuery hooks so changes reflect after mutations invalidate queries
- When TownSettingsEditor saves (useMutation → invalidateQueries), the read-only display refetches and updates
- When a board member is added (in a different route), the checklist item updates on return to dashboard because the query refetches on window focus
- Use typed fetch functions from lib/api/ and mutation hooks from lib/mutations/ throughout

TASK 10: Verification
- Navigate to /dashboard after wizard completion — all wizard-configured data displays correctly

- Click Edit on each settings section — form pre-fills with current values
