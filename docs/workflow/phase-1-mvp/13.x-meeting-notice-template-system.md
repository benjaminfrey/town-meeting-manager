# Session 13.x: Meeting Notice Template System

**Depends on:** Pre-13.01-A (board tabs), Pre-13.01-B (compliance engine)
**See also:** Advisory 3.4 — Meeting Notice Template System & Compliance Engine

---

## Prompt

Build the full meeting notice template system. Each board has its own block-based notice template. Admins configure templates from two places: the `/settings/meeting-notices` overview page and the board detail page Settings tab. The notice generation flow on the AgendaBuilder page is updated to use the template and gate the `noticed` meeting status behind it. The notice compliance engine is wired to the AgendaBuilder and meeting creation flow.

---

## Database Migration

Create `packages/web/supabase/migrations/XXX_notice_template.sql`:

```sql
-- Notice template blocks stored on board
ALTER TABLE board
  ADD COLUMN notice_template_blocks JSONB;
  -- null = not configured

-- Notice tracking on meeting
ALTER TABLE meeting
  ADD COLUMN notice_generated_at TIMESTAMPTZ,
  ADD COLUMN notice_pdf_storage_path TEXT,
  ADD COLUMN notice_published_at TIMESTAMPTZ;
```

Block config panels per type (all use react-hook-form):

- **signature_block**: name field (text input)
- **spacer**: height (select: sm/md/lg)

### Task 6 — Notice generation flow update (AgendaBuilder)

Update `packages/web/src/routes/meetings.$meetingId.agenda.tsx`:

1. **Compliance banner** — Add `NoticeComplianceBanner` component above the "Generate Meeting Notice" button:
   - Calls `getNoticeDeadline()` from the compliance engine with the meeting's date, type, and town's state
   - Renders a colored banner (green/yellow/red/grey) with deadline date, days remaining, and statute citation
   - Advisory message is always shown, even if overdue

2. **Generate Meeting Notice button** — Update to use the board's template:
   - If board has no template configured: show a warning "This board has no notice template configured. [Configure template →]" and disable the button
   - If template exists: call `POST /api/meetings/:meetingId/meeting-notice`
   - On success: show two buttons: "Download PDF" and "Publish to Portal"

3. **Publish to Portal action** — Confirmation dialog showing:
   - The applicable statute and required notice period
   - If overdue: amber warning "The statutory deadline has passed. This notice is advisory — you may still post it."
   - "Publish Notice" confirm button
   - On confirm: sets `meeting.notice_published_at`, transitions `meeting.status` to `noticed`

4. **Status guard** — The "Mark as Noticed" status change (if it exists as a manual action) is replaced entirely by the notice publish flow. Remove any existing manual `noticed` status transition.

### Task 7 — Meeting creation flow — compliance forecast

Update `packages/web/src/routes/setup.tsx` or wherever meeting type is selected during creation:

When the user selects meeting type in the new meeting wizard/dialog:
- Call `forecastEarliestMeetingDate()` for the selected type
- If the selected meeting type has special notice requirements (anything beyond standard 24-hour board meeting):
  - Show an inline callout: "This meeting type requires [X] days notice per [statute]. If you post notice today, the earliest meeting date is [date]."

### Task 8 — API endpoint update

Update `packages/api/src/routes/meetings.ts` (or equivalent):

`POST /api/meetings/:meetingId/meeting-notice`:
1. Fetch the meeting (with board + town)
2. Fetch the board's `notice_template_blocks`
3. If no template: return 422 with `{ error: 'no_template_configured' }`
4. Render the template: replace merge fields, fetch agenda items if `agenda_summary` block present
5. Generate PDF using Puppeteer (matching pattern from 06.03 for agenda packets) — not pdf-lib, since the template may have complex layout
6. Upload to Supabase Storage at `notices/{meetingId}/notice-{timestamp}.pdf`
7. Update `meeting.notice_generated_at` and `meeting.notice_pdf_storage_path`
8. Return `{ pdfUrl, noticeGeneratedAt }`

New endpoint: `POST /api/meetings/preview-notice`:
- Accepts `{ templateBlocks, sampleData }` in body
- Renders and returns a PDF — no DB writes
- Used for the template preview "Generate Sample PDF" button

const { data: boardsWithoutTemplate = 0 } = useQuery({
  queryKey: [...queryKeys.boards.byTown(townId), 'templateStatus'],

      .from('board')
      .select('*', { count: 'exact', head: true })
      .eq('town_id', townId)
