# Session 05.01 — Test Infrastructure Setup

**Phase:** 1 — MVP | **Block:** 05 — Testing -- Foundation
**Dependencies:** 02.02
**Estimated tasks:** 10

---

## Description

Set up the complete testing infrastructure for the monorepo: Vitest with React Testing Library for unit and component tests, React Query test utilities and msw (Mock Service Worker) for component/integration tests, and Playwright for end-to-end browser tests. This session creates reusable mock providers, test utilities, and fixtures that all subsequent test sessions depend on.

## Tasks

1. Install Vitest and configure for the monorepo (vitest.config.ts at root, per-package configs)
2. Install React Testing Library (@testing-library/react, @testing-library/jest-dom, @testing-library/user-event)
3. Create test utilities: React Query test wrapper (QueryClient with retry: false, createWrapper() helper)
4. Create test utilities: auth context mock (mock useCurrentUser returning configurable role, town_id, permissions)
5. Create test utilities: render helper that wraps components in all required providers (QueryClient, Auth, Router, Theme)
6. Set up msw (Mock Service Worker) for mocking Supabase API calls in integration tests
7. Install and configure Playwright for E2E tests (playwright.config.ts with Chrome, Firefox, WebKit)
8. Create Playwright fixtures: authenticated user, seeded town data, clean database state
9. Configure Turborepo test pipeline (turbo run test, turbo run test:e2e)
10. Verify: unit test runs with mock providers, integration test connects to in-memory SQLite, Playwright launches browser
