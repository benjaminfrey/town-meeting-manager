# Session 02.03 — Authentication UI

**Phase:** 1 — MVP | **Block:** 02 — Web Application Foundation
**Dependencies:** 02.02
**Estimated tasks:** 10

---

## Description

Build the authentication UI layer including login, logout, password reset, protected route guards, and the AuthProvider context. Wire Supabase GoTrue authentication to PowerSync initialization so the sync engine only connects after successful login, and disconnect cleanly on logout.

## Tasks

1. Create LoginPage component with email/password form (React Hook Form + Zod validation)
2. Create an AuthProvider context that wraps the app, manages Supabase auth session state
3. Implement login flow: Supabase auth.signInWithPassword → store session → initialize PowerSync → redirect to dashboard
4. Implement logout flow: PowerSync disconnect → Supabase auth.signOut → clear local data → redirect to login
5. Create ProtectedRoute wrapper component that checks auth state and redirects unauthenticated users to /login
6. Create a useCurrentUser() hook that returns the authenticated user's person_id, role, town_id, permissions from JWT claims
7. Handle auth state changes (session refresh, token expiry) with Supabase onAuthStateChange listener
8. Create a ForgotPasswordPage with email input for password reset flow
9. Style login page with shadcn/ui components and town branding placeholder
10. Verify: login works with test user, JWT contains custom claims (town_id, role), PowerSync initializes after auth, protected routes redirect correctly

## Prompt

```
You are building the authentication UI for the Town Meeting Manager web application. The app uses Supabase GoTrue for authentication and PowerSync for offline-first data sync. PowerSync requires a valid Supabase JWT to authorize sync operations.

Read docs/advisory-resolutions/1.3a-frontend-architecture.md, specifically the "Authentication Flow" section:
- Client authenticates with Supabase Auth directly, not through PowerSync
- Supabase issues JWT + refresh token
- PowerSync's SupabaseConnector.fetchCredentials() returns the JWT to authorize sync
- Supabase client handles token refresh automatically via onAuthStateChange
- Use RS256 asymmetric keys (JWKS auto-detect)

Read docs/advisory-resolutions/1.2-roles-permissions.md for the role model:
- 4 roles: sys_admin, admin, staff, board_member
- PERSON entity with user_account, board_members, resident_account
- The JWT should contain custom claims: town_id, role, person_id, permissions
- These claims are set by Supabase Auth hooks (configured server-side in session 01.xx)

PROJECT CONTEXT:
- packages/web with React Router v7 Framework Mode, Tailwind v4, shadcn/ui
- PowerSync + Supabase connector already set up (session 02.02)
- React Hook Form + Zod for form handling
- The SupabaseConnector from 02.02 needs a valid auth session before PowerSync can connect

TASK 1: Create LoginPage component
- Create src/routes/login.tsx (this is the /login route)
- Build a login form with:
  - Email field (type="email", required, validated with Zod: z.string().email())
  - Password field (type="password", required, validated with Zod: z.string().min(8))

TASK 2: Create AuthProvider context
- Create src/providers/AuthProvider.tsx
- The AuthProvider manages the entire auth lifecycle:
  - State: { user: User | null, session: Session | null, isLoading: boolean, isAuthenticated: boolean }
  - On mount, call supabase.auth.getSession() to check for an existing session
  - Subscribe to supabase.auth.onAuthStateChange() for session updates:
    - SIGNED_IN → update user/session state, connect PowerSync
    - SIGNED_OUT → clear user/session state, disconnect PowerSync
    - TOKEN_REFRESHED → update session (PowerSync handles this via fetchCredentials)
    - INITIAL_SESSION → set initial auth state
  - Clean up the onAuthStateChange subscription on unmount
- Create AuthContext with React.createContext
- The provider must wrap PowerSyncProvider — the PowerSync connection should only be established AFTER authentication:
  <AuthProvider>      ← manages auth state
    <PowerSyncProvider> ← connects only when authenticated
      <Routes />
    </PowerSyncProvider>
  </AuthProvider>
- Modify the PowerSyncProvider from session 02.02 to accept an `authenticated` prop or check auth context internally. PowerSync.connect() should only be called when the user is authenticated. When the user logs out, PowerSync.disconnect() should be called.

TASK 3: Implement login flow
- In the LoginPage form onSubmit handler:
  1. Call supabase.auth.signInWithPassword({ email, password })
  2. If error → display error message in the form (use React Hook Form's setError or a form-level error state)
  3. If success → the onAuthStateChange listener in AuthProvider fires SIGNED_IN
  4. AuthProvider updates state → PowerSyncProvider detects auth → calls PowerSync.connect()
  5. Navigate to /dashboard (or /setup if the user has no town_id — first-time admin)
- Handle specific Supabase auth errors:
  - "Invalid login credentials" → show "Invalid email or password"
  - "Email not confirmed" → show "Please check your email to confirm your account"
  - Network error → show "Unable to connect. Please check your internet connection."
- Use React Router v7's useNavigate() for redirect after login
- Determine redirect destination:
  - If user JWT has no town_id claim → redirect to /setup (onboarding wizard)
  - If user JWT has town_id → redirect to /dashboard

TASK 4: Implement logout flow
- Create a logout function in AuthProvider (exposed via context):
  async function logout() {
    1. Disconnect PowerSync: await powerSync.disconnectAndClear()
       - This stops sync AND clears the local SQLite database
       - Critical: local data must be cleared on logout for multi-user device scenarios
    2. Sign out of Supabase: await supabase.auth.signOut()
       - This clears the session from localStorage
    3. Navigate to /login
  }
- Add a logout button to the user menu in the RootLayout header
- The logout button should show a confirmation dialog (shadcn/ui Dialog): "Are you sure you want to sign out? Any unsynced changes will be lost."
- If there are pending uploads (ps_crud count > 0), the dialog should warn: "You have N unsynced changes. Signing out will discard them. Are you sure?"

TASK 5: Create ProtectedRoute wrapper
- Create src/components/ProtectedRoute.tsx
- This component checks auth state from AuthContext:
  - If isLoading → show a loading spinner (full-page centered spinner)
  - If !isAuthenticated → redirect to /login using <Navigate to="/login" replace />
  - If isAuthenticated → render children (or <Outlet />)
- Apply ProtectedRoute to the RootLayout:
  - All routes using RootLayout (/dashboard, /boards/:id, /meetings/:id) are protected
  - /login and /forgot-password are NOT protected (accessible without auth)
  - /setup is protected (must be authenticated) but has a special condition: only accessible if user has no town_id
- Also create a reverse guard for /login: if the user IS authenticated, redirect them away from /login to /dashboard

TASK 6: Create useCurrentUser() hook
- Create src/hooks/useCurrentUser.ts
- This hook extracts user identity and permissions from the Supabase JWT claims:
  interface CurrentUser {
    id: string;              // Supabase auth user ID
    personId: string;        // PERSON entity ID (from JWT claim)
    email: string;
    townId: string | null;   // null if no town set up yet
    role: 'sys_admin' | 'admin' | 'staff' | 'board_member';
    govTitle: string | null; // display title like "Town Clerk"
    permissions: Record<string, boolean>; // JSONB permissions from user_account
  }
- Extract claims from the JWT access token:
  - Use supabase.auth.getSession() to get the current session
  - Decode the JWT payload (the access_token is a standard JWT — decode the payload segment)
  - Custom claims should be at session.user.app_metadata or in the JWT payload under custom claim keys
- If claims are not yet present in the JWT (Supabase Auth hooks not configured), return sensible defaults with TODO comments
- The hook should be reactive: when the session refreshes, the returned user data updates
- Export this hook — it will be used extensively in route guards, permission checks, and UI rendering

TASK 7: Handle auth state changes
- In AuthProvider, ensure the onAuthStateChange listener handles all events:
  - SIGNED_IN: Set user and session state. If PowerSync is not connected, connect it.
  - SIGNED_OUT: Clear state. Disconnect PowerSync. Clear local database.
  - TOKEN_REFRESHED: Update session state. PowerSync handles this internally via fetchCredentials() — no explicit action needed, but update the local session reference.
  - INITIAL_SESSION: On app load, this fires with the restored session (if any). Set initial state.
  - PASSWORD_RECOVERY: Redirect to a password reset form (handle in a future session).
- Handle the edge case where the refresh token expires (user has been offline too long):
  - Supabase will fire SIGNED_OUT
  - Show a toast notification: "Your session has expired. Please sign in again."
  - Redirect to /login
- Use the shadcn/ui Toast component (from the Toaster setup in 02.01) for session-related notifications
```
