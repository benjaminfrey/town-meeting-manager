# Session 15.01 — Phase 1 Polish: Error Handling, Loading States, Edge Cases

**Phase:** 1 — MVP | **Block:** 15 — Deployment & Polish
**Dependencies:** 14.01
**Estimated tasks:** 10

---

## Description

Conduct a comprehensive polish pass across the entire Phase 1 application, adding missing loading states, error handling, and edge case coverage. This includes Suspense boundaries with skeleton loaders, a global error boundary, toast notifications for user feedback, connection resilience handling, keyboard navigation for accessibility, and resolution of any remaining console errors, TypeScript warnings, or ESLint issues. The goal is production readiness.

## Tasks

1. Audit all pages for loading states: add Suspense boundaries with skeleton loaders for data-dependent content
2. Audit all forms for error handling: network errors, validation errors, conflict errors from sync
3. Create global error boundary with friendly error page and "Report Issue" action
4. Implement toast notification system for success/error feedback (using shadcn/ui Toast)
5. Handle edge cases: empty boards (no members), empty meetings (no agenda), concurrent edits during meeting
6. Implement connection error handling: display Supabase Realtime disconnection warnings, retry logic for failed mutations
7. Add loading spinners and disabled states for async operations (PDF generation, email sending)
8. Implement keyboard navigation and focus management for accessibility (WCAG 2.1 AA)
9. Review and fix any console errors, TypeScript warnings, or ESLint issues
10. Verify: all pages have loading states, errors are handled gracefully, keyboard navigation works, no console errors

TASK 5: Handle edge cases
- Empty states:
  - Board with no members: show "No members yet. Add members to schedule meetings." on the board detail page. Prevent meeting creation until at least 3 members exist (quorum requirement).
  - Meeting with no agenda: show "This meeting has no agenda items. Add items to build the agenda." Prevent agenda publishing with zero items.
  - Board with no meetings: show "No meetings scheduled. Create your first meeting." on the board detail page.
  - Minutes with all "None" sections: valid state (e.g., a very short meeting), but show a notice: "These minutes contain no substantive items. Is this correct?"
  - User with no boards: board member who hasn't been assigned to any board yet — show "You haven't been assigned to any boards. Contact your administrator."
- Concurrent edit handling:
  - During a live meeting, if two operators are running the same meeting (an error, but possible):
    - Supabase Realtime syncs changes between operators in real-time
    - Add a banner: "Another operator is active on this meeting. Changes from both operators will be saved."
  - If two admins edit the same agenda simultaneously:
    - Last write wins (Supabase default) but show a conflict indicator: "This agenda was modified by [other user] at [time]. Refresh to see their changes."
- Boundary conditions:
  - Meeting with maximum agenda items (stress test with 50+ items)
  - Board with maximum members (25 members as configured in the add board form)
  - Very long motion text (500+ characters)
  - Town name with special characters (apostrophes, hyphens)
  - Meeting dates across timezones (all dates should use the town's local timezone — Eastern for Maine)

TASK 6: Implement connection error handling
- Supabase Realtime and mutation errors need user-facing treatment:
  - Mutation failures (write rejected by the server):
    - useMutation's onError callback fires — show an error toast with the failure reason
    - For validation errors (RLS policy denied): "You don't have permission to make this change."
    - For network errors: "Connection lost. Please check your network and try again."
    - TanStack Query automatically retries failed queries up to the configured retry count (default: 2)
  - Realtime disconnection:
    - The ConnectionStatusBar already handles this ("Disconnected" / "Connecting" status)
    - Add a manual "Reconnect" button in the ConnectionStatusBar that calls channel.subscribe() again
  - Stale data on reconnect:
    - When the Realtime connection is restored, call queryClient.invalidateQueries() to refetch all stale data
    - Show a brief "Refreshing data..." indicator while refetching
- Create a connection error handler:
  - File: packages/web/src/lib/connection-error-handler.ts
  - Subscribe to Supabase Realtime connection state changes
  - On disconnect: set ConnectionStatusBar to "Disconnected" state
  - On reconnect: invalidate all queries, set ConnectionStatusBar to "Connected"
  - Categorize mutation errors: retriable (network timeout) vs. permanent (validation failure, permission denied)
  - For permanent errors: surface the specific error message via toast

TASK 8: Implement keyboard navigation and focus management
- WCAG 2.1 AA requires full keyboard operability:
  - All interactive elements must be reachable via Tab key
  - All actions must be triggerable via Enter or Space key
  - Focus must be visible with a clear focus indicator (2px solid outline, high contrast)
  - Focus must be managed during navigation and dialog interactions
- Audit and fix keyboard navigation:
  - Skip-to-content link at the top of every page (already in the portal layout — ensure it's also in the main app layout)
  - Tab order follows visual order (no tabIndex manipulation that breaks the flow)
  - Dialogs trap focus (shadcn/ui Dialog handles this via Radix — verify it works)
  - When a dialog closes, focus returns to the element that opened it
  - When a toast appears, it does not steal focus from the current element
  - Dropdown menus are navigable with arrow keys (shadcn/ui DropdownMenu handles this — verify)
- Focus management for specific flows:
  - Agenda builder: after adding a new item, focus moves to the new item's title field
  - Meeting operations: after recording a motion, focus moves to the next action button
  - Minutes editor: after saving, focus remains in the editor (no page scroll)
  - Form submission errors: focus moves to the first field with an error
- Ensure all custom interactive components (not from shadcn/ui) have proper ARIA attributes:
  - role, aria-label, aria-expanded, aria-selected, aria-disabled as appropriate
  - ConnectionStatusBar: role="status", aria-live="polite" (announces status changes to screen readers)
