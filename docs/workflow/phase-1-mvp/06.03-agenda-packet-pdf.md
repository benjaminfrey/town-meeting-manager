# Session 06.03 — Agenda Packet Generation (Puppeteer)

**Phase:** 1 — MVP | **Block:** 06 — Agenda Building
**Dependencies:** 06.02
**Estimated tasks:** 10

---

## Description

Build the document generation pipeline for agenda packets (Puppeteer) and meeting notices (pdf-lib). This includes the Fastify API server scaffold, Docker configuration for Puppeteer, HTML/CSS templates with Handlebars for agenda packet layout, and the API endpoints that generate PDFs and store them in Supabase Storage.

## Tasks

1. Create packages/api directory setup: Fastify server scaffold with TypeScript, routes, plugins
2. Create Docker container configuration for Puppeteer (memory-limited, single browser instance)
3. Add Puppeteer service to docker-compose.yml (or run within the Fastify API container)
4. Create HTML/CSS template for agenda packet using Handlebars: cover page (town name, board, date, location), table of contents, agenda sections with items, exhibits appendix
5. Use semantic HTML (h1, h2, table, thead, th, article) for WCAG 2.1 AA accessible tagged PDFs
6. Implement Puppeteer PDF generation: launch browser -> new page -> set HTML content -> page.pdf() with headers/footers/page numbers
7. Create agenda packet API endpoint: POST /api/meetings/:meetingId/agenda-packet -> generates PDF -> stores in Supabase Storage -> returns URL
8. Create meeting notice generation using pdf-lib (simple text: date, time, location, board name)
9. Create "Generate Agenda Packet" and "Generate Meeting Notice" buttons on the AgendaBuilder page
10. Verify: agenda packet PDF generates with correct formatting, page numbers, exhibits appendix, meeting notice generates as simple PDF

TASK 2: Create Docker container configuration for Puppeteer
- Create packages/api/Dockerfile:

  - Set PUPPETEER_EXECUTABLE_PATH to the system Chromium path
  - Expose port 3001
- Create packages/api/.dockerignore: node_modules, dist, .turbo

TASK 3: Add Puppeteer service to docker-compose.yml
- Create docker-compose.yml at the repo root (or update existing):
  services:
    api:
      build: ./packages/api
      ports: ["3001:3001"]
      environment:
        - SUPABASE_URL=${SUPABASE_URL}
        - SUPABASE_SERVICE_ROLE_KEY=${SUPABASE_SERVICE_ROLE_KEY}
        - PORT=3001
      deploy:
        resources:
          limits:
            memory: 1G
      restart: unless-stopped
- The API container runs Puppeteer internally — no separate Puppeteer container needed
- Memory limit of 1G provides headroom for Chromium + Node + Fastify
- The single browser instance pattern (Task 6) keeps memory usage bounded

TASK 4: Create Handlebars agenda packet template
- Install Handlebars: pnpm --filter @town-meeting/api add handlebars
- Create packages/api/src/templates/agenda-packet.hbs:
  - Cover page: town name (h1), board name (h2), meeting date and time, location, meeting type
  - If town seal URL exists, include the town seal image
  - Table of contents: list of sections with page references (use CSS counters)
  - For each section:
    - Section heading (h2) with section number
    - For each item in the section:
      - Item heading (h3) with item reference (e.g., "6A")
      - If Item Commentary enabled:
        - STAFF RESOURCE line
        - BACKGROUND section (rendered from markdown to HTML)
        - EXHIBITS numbered list with titles
        - RECOMMENDATION section
        - POSSIBLE MOTION in italic
      - Sub-items listed with sub-numbering (h4)
  - Exhibits appendix: if any exhibits are attached, create an appendix page listing all exhibits by section
  - Footer on each page: "Page X of Y" and the meeting date
- Create packages/api/src/templates/agenda-packet.css:
  - Professional typography: Georgia or Times New Roman for body, Arial/Helvetica for headings
  - @page CSS for page size (Letter 8.5x11), margins (1 inch)
  - Page break rules: section headings avoid page breaks (break-inside: avoid)
  - Table styles for financial warrant data
  - Print-friendly colors (no backgrounds, dark text)

TASK 5: Use semantic HTML for WCAG 2.1 AA accessibility
- The Handlebars template MUST use semantic HTML elements:
  - <h1> for town/board name (document title)
  - <h2> for section headings
  - <h3> for agenda item titles
  - <h4> for sub-item titles
  - <table>, <thead>, <th>, <tbody>, <td> for tabular data (financial warrants, vote tallies)
  - <article> for each agenda item's commentary block
  - <nav> for the table of contents
  - <main> for the primary content area
  - <aside> for supplementary notes
  - <ol> and <ul> for ordered and unordered lists
  - <strong> and <em> for emphasis (not <b> and <i>)
- This ensures Puppeteer produces tagged PDFs that screen readers can navigate
- Include lang="en" on the <html> element
- Include role attributes where semantic elements are insufficient
- Add aria-label to navigation elements

TASK 6: Implement Puppeteer PDF generation service
- File: packages/api/src/services/puppeteer.ts
- Install Puppeteer: pnpm --filter @town-meeting/api add puppeteer
- Implement a singleton browser manager:
  - Launch browser ONCE on server startup with: puppeteer.launch({ headless: true, args: ['--no-sandbox', '--disable-setuid-sandbox', '--disable-dev-shm-usage', '--single-process'] })
  - Expose a generatePdf(html: string, options: PdfOptions) method
  - Method creates a new page (not a new browser), sets HTML content, generates PDF, closes the page
  - Use page.setContent(html, { waitUntil: 'networkidle0' }) to ensure all images/styles load
  - Use page.pdf() with options:
    - format: 'Letter'
    - printBackground: true
    - margin: { top: '1in', bottom: '1in', left: '1in', right: '1in' }
    - displayHeaderFooter: true
    - headerTemplate: '<div style="font-size:9px; text-align:center; width:100%;">{{boardName}} — {{meetingDate}}</div>'
    - footerTemplate: '<div style="font-size:9px; text-align:center; width:100%;"><span class="pageNumber"></span> of <span class="totalPages"></span></div>'
  - Return the PDF as a Buffer
- Handle browser crashes gracefully: if the browser disconnects, relaunch it
- Add a health check method: isBrowserAlive() that checks browser.isConnected()
- Log PDF generation timing for performance monitoring

TASK 7: Create agenda packet API endpoint
- File: packages/api/src/routes/documents.ts
- POST /api/meetings/:meetingId/agenda-packet
  - Request body: { meetingId: string } (meetingId also in URL params)
  - Authentication: validate the request comes from an authenticated user with A6 (generate_agenda_packet) permission
  - Process:
    1. Query meeting data from Supabase directly (the API uses the Supabase admin client — full JOIN support)
    2. Query all agenda items, exhibits, and board/town data for this meeting
    3. Compile the Handlebars template with the meeting data context
    4. Inject the CSS into a <style> tag in the HTML
    5. Call puppeteerService.generatePdf(html, options) to produce the PDF buffer
    6. Upload the PDF to Supabase Storage: bucket "documents", path "{town_id}/meetings/{meeting_id}/agenda-packet-{timestamp}.pdf"
    7. Return the response: { url: string, fileSize: number, generatedAt: string }
  - Error handling: return 400 for invalid meeting, 403 for unauthorized, 500 for PDF generation failure
  - Set timeout: 30 seconds maximum for PDF generation (agenda packets can be long)
- Install Supabase server client: pnpm --filter @town-meeting/api add @supabase/supabase-js
- Use the SUPABASE_SERVICE_ROLE_KEY for server-side access (bypasses RLS for reading meeting data)

TASK 8: Create meeting notice generation with pdf-lib
- Install pdf-lib: pnpm --filter @town-meeting/api add pdf-lib
- File: packages/api/src/services/pdf-lib.ts
- Create generateMeetingNotice(data: MeetingNoticeData): Promise<Buffer>
  - MeetingNoticeData: { townName, boardName, meetingDate, meetingTime, location, virtualLink? }
  - Generate a simple one-page PDF:
    - Town name centered at top (bold, 16pt)
    - "NOTICE OF PUBLIC MEETING" heading centered (bold, 14pt)
    - Body text: "The [Board Name] of the [Town Name] will hold a [regular/special] meeting on [date] at [time] at [location]."
    - If virtual link: "This meeting will also be available via [link]."
    - Footer: "Posted: [current date]"
  - Use pdf-lib's PDFDocument.create(), embed standard fonts (Helvetica, Times-Roman)
  - Return the PDF as a Buffer
- POST /api/meetings/:meetingId/meeting-notice endpoint:
  - Query meeting data from Supabase
  - Call generateMeetingNotice() with the meeting data
  - Upload to Supabase Storage: "{town_id}/meetings/{meeting_id}/meeting-notice-{timestamp}.pdf"
  - Return { url, fileSize, generatedAt }

TASK 9: Create generation buttons on the AgendaBuilder page
