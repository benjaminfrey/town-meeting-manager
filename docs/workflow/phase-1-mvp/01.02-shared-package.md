# Session 01.02 — Shared Package Scaffold

**Phase:** 1 — MVP | **Block:** 01 — Infrastructure & Monorepo Foundation
**Dependencies:** 01.01
**Estimated tasks:** 14

---

## Description

Build out the shared package with TypeScript type definitions, Zod validation schemas, enum constants, and utility functions for all core entities in the Town Meeting Manager data model. This package is consumed by web, mobile, and api packages and serves as the single source of truth for data shapes and validation rules.

## Tasks

1. Set up packages/shared/src/ directory structure: types/, schemas/, constants/, utils/
2. Create TypeScript type definitions for PERSON entity (Person, UserAccount, BoardMember, ResidentAccount)
3. Create types for TOWN, BOARD, BOARD_MEMBERSHIP
4. Create types for MEETING, AGENDA_ITEM, AGENDA_TEMPLATE
5. Create types for MOTION, VOTE_RECORD, ATTENDANCE
6. Create types for MINUTES_DOCUMENT, MINUTES_SECTION
7. Create enum constants: UserRole, MeetingStatus, MotionType, VoteType, MinutesStyle, MeetingFormality, BoardType, MunicipalityType
8. Create Zod validation schemas for all core entities
9. Create permission action constants matching the 30-action permissions matrix from advisory 1.2
10. Create utility functions: generateId (UUID v7), formatGovernmentTitle, calculateQuorum
11. Create shared PowerSync schema definitions (TableV2) for all core tables
12. Set up package exports in package.json (types, schemas, constants, utils)
13. Configure TypeScript build with declarations
14. Verify package builds and types export correctly

## Prompt

```
You are building the shared package (@town-meeting/shared) for the Town Meeting Manager monorepo. This package contains TypeScript types, Zod validation schemas, constants, and utilities shared across web, mobile, and API packages.

Read docs/advisory-resolutions/1.2-roles-permissions.md for the PERSON entity architecture, role definitions, and the 30-action permissions matrix. Also read docs/advisory-resolutions/2.2-ai-minutes-architecture.md for the minutes document structure. Here is essential context inline for reference:

PERSON ENTITY ARCHITECTURE (from advisory 1.2):
- PERSON is the identity anchor: name, email (unique per town), created_at, archived_at
- PERSON has 0-or-1 user_account (role: sys_admin | admin | staff | board_member)
- PERSON has 0-to-many board_members (board_id, seat_title, term_start, term_end, status, is_default_rec_sec)
- PERSON has 0-or-1 resident_account (subscriptions, notification_preferences)
- user_account has: role, gov_title (display label), permissions (JSONB matrix), auth_user_id
- staff and board_member roles are mutually exclusive on the same PERSON

FOUR ROLES:
- sys_admin: system infrastructure, not a meeting user
- admin: full control, manages users/boards/settings, also a regular user
- staff: configurable permissions per account via matrix, per-board with global defaults
- board_member: view agenda, upload files for review, submit questions/comments (flow through admin)

PERMISSIONS MATRIX (30 actions across 6 areas):
- Town & System Management: T1-T4 (admin only)
- Agenda & Meeting Prep: A1-A7
- Live Meeting Operations: M1-M8
- Minutes & Records: R1-R6
- Civic Engagement: C1-C5
- View & Download: V1-V5 (always allowed for authenticated users)

TASK 10: Create utility functions

  Install: `pnpm --filter @town-meeting/shared add uuidv7`

- quorumAfterRecusal(presentCount: number, recusalCount: number, totalMembers: number): { hasQuorum: boolean; participatingCount: number; quorumNeeded: number }

TASK 11: Create PowerSync schema definitions (packages/shared/src/powersync/schema.ts)
Install PowerSync common package: `pnpm --filter @town-meeting/shared add @powersync/common`
Define PowerSync TableV2 schemas for all core tables. These define the local SQLite schema that PowerSync uses on client devices:
```typescript
import { column, Schema, TableV2 } from "@powersync/common";

const town = new TableV2({ /* columns matching the Town type */ });
const person = new TableV2({ /* columns matching the Person type */ });
const user_account = new TableV2({ /* ... */ });
// ... all core tables

export const AppSchema = new Schema({ town, person, user_account, board_member, board, meeting, agenda_item, motion, vote_record, meeting_attendance, minutes_document, minutes_section, agenda_template });
```
Note: PowerSync TableV2 column types are: column.text, column.integer, column.real. JSONB fields use column.text (serialized JSON string on the client).
