# Session 02.01 — React Router v7 + Tailwind v4 Web App Setup

**Phase:** 1 — MVP | **Block:** 02 — Web Application Foundation
**Dependencies:** 01.02
**Estimated tasks:** 13

---

## Description

Initialize the packages/web workspace with React Router v7 in Framework Mode, Tailwind CSS v4 with CSS-first configuration, and shadcn/ui component library. Establish the route structure, layout shell, error boundaries, and theming foundation that all subsequent web sessions build upon.

## Tasks

1. Initialize packages/web with React Router v7 Framework Mode (use create-react-router or manual setup)
2. Configure TypeScript for web package (extending tsconfig.base.json)
3. Install and configure Tailwind CSS v4 (CSS-first: @import "tailwindcss" in app.css, no tailwind.config.js)
4. Install and initialize shadcn/ui (npx shadcn@latest init, configure components.json)
5. Install initial shadcn/ui components: Button, Dialog, Form, Input, Select, Tabs, Toast, Badge, Card, Sheet
6. Set up the app layout structure: root layout with sidebar navigation, header, main content area
7. Create route structure: /login, /setup (wizard), /dashboard, /boards/:boardId, /meetings/:meetingId
8. Install and configure react-error-boundary with feature-area boundaries
9. Create a ThemeProvider context (light/dark mode using Tailwind CSS v4 dark mode)
10. Set up path aliases in tsconfig (@/ for src/)
11. Add @town-meeting/shared as a workspace dependency
12. Configure Vite dev server proxy for Supabase API endpoints
13. Verify: dev server starts, routes render, shadcn/ui components display correctly, shared package types import

You are setting up the React web application for the Town Meeting Manager project — a municipal meeting management platform for Maine towns. This is a pnpm monorepo; you are working in packages/web.

Read docs/advisory-resolutions/1.3a-frontend-architecture.md for the confirmed frontend architecture decisions. Key decisions from that document that apply here:
- React Router v7 Framework Mode with clientLoader/clientAction (NOT server loaders — data comes from Supabase via TanStack Query)
- Tailwind CSS v4 with CSS-first configuration (@import "tailwindcss", no tailwind.config.js)
- shadcn/ui components (Radix UI primitives, copy-paste model into components/ui/)
- react-error-boundary for feature-area error boundaries
- TanStack Query for server state (configured in session 02.02); no Redux/Zustand needed

PROJECT CONTEXT:
