## Description

Create the production deployment configuration for the self-hosted Town Meeting Manager platform. This includes Docker Compose for all services (Supabase stack, PowerSync, Fastify API, Nginx, Puppeteer), Nginx reverse proxy with wildcard SSL, automated PostgreSQL backups, health monitoring, deployment scripts, and step-by-step deployment documentation. The deployment targets a dedicated server handling 3 towns for beta testing, per advisory 3.2.

## Tasks

1. Create production docker-compose.yml with all services (Supabase stack, PowerSync, Fastify API, Nginx, Puppeteer)
2. Configure Nginx reverse proxy: app.townmeetingmanager.com → web app, api.townmeetingmanager.com → Fastify, *.townmeetingmanager.com → public portal, supabase.townmeetingmanager.com → Studio (restricted)
3. Set up Let's Encrypt SSL with certbot for wildcard certificates (*.townmeetingmanager.com)
4. Create production environment variable template (.env.production.example) with all required secrets
5. Configure PostgreSQL automated backups: daily pg_dump with WAL archiving, 30-day retention
6. Create backup restore procedure and test it
7. Set up basic health check monitoring for all Docker services (container health checks, restart policies)
8. Create deployment scripts: build, migrate, deploy, rollback
9. Configure Turborepo production build pipeline: turbo run build → output to dist/
10. Create a deployment documentation file with step-by-step instructions
11. Verify: production build succeeds, docker-compose starts all services, SSL certificates work, backups run, health checks pass

TASK 2: Configure Nginx reverse proxy
- File: infrastructure/nginx/nginx.conf
- Main server blocks:
  - **app.townmeetingmanager.com** → serve static web app files (Vite build output):
    ```nginx
    server {
        listen 443 ssl;
        server_name app.townmeetingmanager.com;
        ssl_certificate /etc/letsencrypt/live/townmeetingmanager.com/fullchain.pem;
        ssl_certificate_key /etc/letsencrypt/live/townmeetingmanager.com/privkey.pem;

        root /var/www/web;
        index index.html;

        location / {
            try_files $uri $uri/ /index.html;
        }

        location /health {
            return 200 "ok";
        }
    }
    ```
  - **api.townmeetingmanager.com** → proxy to Fastify API:
    ```nginx
    server {
        listen 443 ssl;
        server_name api.townmeetingmanager.com;
        # SSL config same as above

        location / {
            proxy_pass http://api:3001;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }

        location /api/webhooks/ {
            # Postmark webhooks — no rate limiting
            proxy_pass http://api:3001;
        }
    }
    ```
  - **\*.townmeetingmanager.com** → public portal (same web app with subdomain header):
    ```nginx
    server {
        listen 443 ssl;
        server_name *.townmeetingmanager.com;
        # SSL config with wildcard cert

        root /var/www/web;
        index index.html;

        location / {
            try_files $uri $uri/ /index.html;
            add_header X-Town-Subdomain $subdomain;
        }

        location /sitemap.xml {
            proxy_pass http://api:3001/api/portal/sitemap;
        }

        location /robots.txt {
            proxy_pass http://api:3001/api/portal/robots;
        }

        set $subdomain "";
        if ($host ~ ^(.+)\.townmeetingmanager\.com$) {
            set $subdomain $1;
        }
    }
    ```
  - **supabase.townmeetingmanager.com** → Supabase Studio (restricted):
    ```nginx
    server {
        listen 443 ssl;
        server_name supabase.townmeetingmanager.com;
        # SSL config

        # Restrict to specific IPs or require basic auth
        allow 10.0.0.0/8;    # Internal network
        allow YOUR_ADMIN_IP;  # Admin's IP address
        deny all;

        location / {
            proxy_pass http://supabase-studio:3000;
        }
    }
    ```
- HTTP → HTTPS redirect for all domains:
  ```nginx
  server {
      listen 80;
      server_name *.townmeetingmanager.com townmeetingmanager.com;
      return 301 https://$host$request_uri;
  }

TASK 5: Configure PostgreSQL automated backups
- File: infrastructure/scripts/backup.sh
- Backup strategy per advisory 3.2: daily pg_dump with WAL archiving, 30-day retention
- Daily backup script:
  ```bash
  #!/bin/bash
  TIMESTAMP=$(date +%Y%m%d_%H%M%S)
  BACKUP_DIR="${BACKUP_PATH:-/var/backups/postgres}"
  RETENTION_DAYS="${BACKUP_RETENTION_DAYS:-30}"

  # Create backup directory
  mkdir -p "$BACKUP_DIR"

  # Full database dump (custom format for selective restore)
  docker compose -f infrastructure/docker-compose.production.yml exec -T supabase-db \
    pg_dump -U postgres -Fc -Z 9 town_meeting_manager \
    > "$BACKUP_DIR/tmm_${TIMESTAMP}.dump"

  # Also create a plain SQL backup for emergency recovery
  docker compose -f infrastructure/docker-compose.production.yml exec -T supabase-db \
    pg_dump -U postgres --clean --if-exists town_meeting_manager \
    | gzip > "$BACKUP_DIR/tmm_${TIMESTAMP}.sql.gz"

  # Delete backups older than retention period
  find "$BACKUP_DIR" -name "tmm_*.dump" -mtime +$RETENTION_DAYS -delete
  find "$BACKUP_DIR" -name "tmm_*.sql.gz" -mtime +$RETENTION_DAYS -delete

  # Log backup size and status
  BACKUP_SIZE=$(du -sh "$BACKUP_DIR/tmm_${TIMESTAMP}.dump" | cut -f1)
  echo "[$(date)] Backup completed: tmm_${TIMESTAMP}.dump ($BACKUP_SIZE)"
  ```
- Configure WAL archiving for point-in-time recovery:
  - Set PostgreSQL parameters in the Supabase postgres config:
    - wal_level = replica
    - archive_mode = on
    - archive_command = 'cp %p /var/backups/wal/%f'
  - WAL files enable point-in-time recovery between daily backups
- Schedule the backup via cron:
  ```
  0 2 * * * /opt/town-meeting-manager/infrastructure/scripts/backup.sh >> /var/log/tmm-backup.log 2>&1
  ```
  - Runs at 2:00 AM daily (low-activity time for Maine town halls)

TASK 6: Create backup restore procedure
- File: infrastructure/scripts/restore.sh
- Restore from a custom-format dump:
  ```bash
  #!/bin/bash
  BACKUP_FILE=$1

  if [ -z "$BACKUP_FILE" ]; then
    echo "Usage: ./restore.sh <backup_file.dump>"
    echo "Available backups:"
    ls -la ${BACKUP_PATH:-/var/backups/postgres}/tmm_*.dump
    exit 1
  fi

  echo "WARNING: This will replace the current database with the backup."
  echo "Backup file: $BACKUP_FILE"
  read -p "Are you sure? (yes/no): " CONFIRM

  if [ "$CONFIRM" != "yes" ]; then
    echo "Restore cancelled."
    exit 0
  fi

  # Stop the application services (keep database running)
  docker compose -f infrastructure/docker-compose.production.yml stop api powersync

  # Restore the database
  docker compose -f infrastructure/docker-compose.production.yml exec -T supabase-db \
    pg_restore -U postgres -d town_meeting_manager --clean --if-exists < "$BACKUP_FILE"

  # Restart all services
  docker compose -f infrastructure/docker-compose.production.yml up -d

  echo "Restore completed. Services restarted."
  ```

  ```bash
  #!/bin/bash
  set -e

  PREVIOUS_COMMIT=$1

  if [ -z "$PREVIOUS_COMMIT" ]; then
    echo "Usage: ./rollback.sh <commit_hash>"
    echo "Recent commits:"
    git log --oneline -10
    exit 1
  fi

  echo "Rolling back to commit: $PREVIOUS_COMMIT"

  # Checkout the previous version

  # Rebuild and redeploy
  pnpm install --frozen-lockfile
  pnpm turbo run build
  cp -r packages/web/dist/* /var/www/web/

  docker compose -f infrastructure/docker-compose.production.yml build api puppeteer
