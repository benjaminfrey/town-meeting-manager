# Session 06.01 — Agenda Template System

**Phase:** 1 — MVP | **Block:** 06 — Agenda Building
**Dependencies:** 04.02
**Estimated tasks:** 11

---

## Description

Build the agenda template system that allows admins to define reusable meeting structures per board. Templates contain ordered sections with configurable types (ceremonial, procedural, action, discussion, etc.), minutes behavior defaults, and fixed/removable flags. This is the foundation for agenda building and the AI minutes pipeline.

## Tasks

1. Create AgendaTemplateListPage at /boards/:boardId/templates showing templates for the board
2. Create AgendaTemplateEditor component: visual editor for defining agenda template sections
3. Implement fixed section types: Call to Order, Pledge of Allegiance, Approval of Minutes, Adjournment — these appear in every meeting from this template
4. Implement repeating section types: Old Business, New Business, Public Comment, Executive Session — these define where agenda items can be added
5. Implement custom section types: user-defined sections with custom titles
6. Create drag-and-drop reordering for template sections (sort_order)
7. Create a default template per board (auto-created, marked is_default)
8. Implement template cloning (duplicate an existing template as starting point)
9. Create template preview showing how a meeting agenda would look with this template
10. Store templates as JSONB in AGENDA_TEMPLATE table with board_id scope
11. Verify: create template, add/remove/reorder sections, set default, clone template, preview renders

## Prompt

```
You are building the Agenda Template System for the Town Meeting Manager — a municipal meeting management platform for Maine towns. Templates define the reusable section structure that boards use for their meetings.

Read docs/advisory-resolutions/2.2-ai-minutes-architecture.md Section 1 (Agenda Templates) and Section 7 (Agenda Template System — Full Specification) for the template data model, section types, and minutes behavior enums. Key decisions from that document:
- Advisory Q12: all fixed sections are always included in the minutes (show "None" if empty)
- Advisory Q1: Item Commentary fields are template-level defaults (shown/hidden per section)
- Advisory Q13: motion display format is board-level configurable (block_format or inline_narrative)

PROJECT CONTEXT:
- Monorepo: packages/web (React 19 + React Router v7 Framework Mode), packages/shared (types, schemas)
- PowerSync for offline-first local SQLite — use @powersync/react hooks (useQuery, useSuspenseQuery)
- Kysely ORM via @powersync/kysely-driver for type-safe SQL
- Tailwind CSS v4 + shadcn/ui (Radix UI) for components
- React Hook Form + Zod for form validation
- clientLoader pattern: load from PowerSync before route renders, then useQuery for reactivity

TEMPLATE DATA MODEL (from advisory 2.2):
Each template section has these properties:
- title: string — section heading (e.g., "Official Action Items")
- sort_order: number — position in the agenda
- section_type: enum — ceremonial, procedural, minutes_approval, financial, public_input, report, action, discussion, public_hearing, executive_session, other
- minutes_behavior: enum — skip, timestamp_only, action_only, summarize, full_record
- is_fixed: boolean — if true, appears on every agenda from this template and cannot be removed
- description: string | null — default description text shown under the heading
- default_items: array — pre-populated items that appear each time (e.g., "Payroll Warrants #___")
- show_item_commentary: boolean — whether to show background/recommendation/suggested_motion fields

TASK 1: Create AgendaTemplateListPage
- Route: /boards/:boardId/templates
- File: packages/web/src/routes/boards.$boardId.templates.tsx
- Use clientLoader to load the board record and all AGENDA_TEMPLATE records for this board from PowerSync
- Display a DataTable with columns: template name, section count, is_default (Badge), last updated, actions (edit, clone, delete)
- "Create Template" button opens a dialog for naming the new template
- Each board auto-creates a default template if none exists (see Task 7)
- The default template cannot be deleted (only renamed or modified)

TASK 2: Create AgendaTemplateEditor component
- File: packages/web/src/components/agenda/AgendaTemplateEditor.tsx
- Full-page editor loaded when clicking "Edit" on a template from the list
- Route: /boards/:boardId/templates/:templateId/edit
- Left panel: ordered list of sections with drag handles, section type icons, and fixed/removable badges
- Right panel: section detail editor (appears when a section is selected)
- Section detail fields: title (text input), section_type (Select dropdown), minutes_behavior (Select dropdown), is_fixed (Checkbox), description (Textarea), show_item_commentary (Checkbox)
- "Add Section" button at the bottom of the section list with a popover for choosing section_type
- "Remove Section" button on non-fixed sections (with confirmation Dialog for sections that have default_items)
- Template name is editable inline at the top of the editor
- Save writes the entire template JSONB to PowerSync local SQLite via powersync.execute()

TASK 3: Implement fixed section types
- Fixed sections are marked is_fixed: true in the template
- Fixed sections cannot be removed from the template or from agendas built from this template
- Fixed sections display a locked icon and a "Fixed" Badge in the section list
- Standard fixed sections for the default Select Board template:
  1. Call to Order [procedural, timestamp_only, fixed]
  2. Amendments to the Agenda [procedural, action_only, fixed]
  3. Minutes of Previous Meeting(s) [minutes_approval, action_only, fixed]
  4. Public Comments [public_input, summarize, fixed]
  5. Fiscal Warrants [financial, action_only, fixed]
  6. New Business [action, summarize, fixed]
  7. Adjournment [procedural, timestamp_only, fixed]
- The admin CAN edit the title and settings of fixed sections (e.g., rename "Public Comments" to "Citizen Comments & Correspondence"), but cannot remove them

TASK 4: Implement repeating section types
- Repeating sections define containers where agenda items can be added per meeting
- They are NOT fixed — the admin can remove them from a template if their board does not use them
- Standard repeating sections:
  - Presentations and Special Guests [report, summarize]
  - Unfinished Business / Old Business [action, summarize]
  - Town Manager Report [report, summarize]
  - Discussion Items [discussion, summarize]
  - Future Agenda Items [discussion, summarize]
  - Executive Session [executive_session, full_record]
- Each repeating section supports default_items — pre-populated items that appear each time a meeting is created from this template (e.g., "Payroll Warrants #___" under Fiscal Warrants)

TASK 5: Implement custom section types
- The "Add Section" button includes an "Other / Custom" option at the bottom of the section_type picker
- Custom sections get section_type: "other" and an editable title
- Custom sections can be set as fixed or non-fixed
- Custom sections support the same minutes_behavior settings as standard sections
- Use case: a board might add a "Committee Reports" or "Liaison Reports" section

TASK 6: Create drag-and-drop reordering
- Use a drag-and-drop library (dnd-kit recommended — lightweight, accessible, works with Radix/shadcn)
- Install: pnpm --filter @town-meeting/web add @dnd-kit/core @dnd-kit/sortable @dnd-kit/utilities
- Sections can be reordered by dragging — sort_order updates for all affected sections
- Call to Order should remain first and Adjournment should remain last by convention, but the system should not enforce this rigidly (some boards may have Pledge of Allegiance before Call to Order)
- Drag handles use a grip icon on the left side of each section row
- Reordering writes the updated sort_order values to PowerSync immediately

TASK 7: Create default template per board
- When a board is created (or when the templates page is first visited and no templates exist), auto-create a default template
- The default template name: "Standard [Board Name] Meeting"
- Default template is marked is_default: true — only one template per board can be default
- Setting another template as default removes the flag from the previous default
- Default template is pre-populated with the standard sections for the board type:
  - Select Board / Town Council: 12-section template from advisory 2.2 Section 7
  - Planning Board: 6-section template from advisory 2.2 Section 11
  - Other board types: a minimal template with Call to Order, Old Business, New Business, Adjournment

TASK 8: Implement template cloning
- "Clone" action on each template in the list
- Opens a Dialog to enter the new template name (default: "[Original Name] (Copy)")
- Creates a deep copy of all sections and default_items
- The cloned template is NOT set as default
- Clone writes to PowerSync as a new AGENDA_TEMPLATE record

TASK 9: Create template preview
- "Preview" button in the template editor
- Opens a Dialog or Sheet showing the template rendered as it would appear on a meeting agenda
- Preview displays:
  - Board name and placeholder date/time/location
  - Each section heading in order with its section_type icon
  - Default items listed under their sections
  - Fixed sections marked with a subtle indicator
  - Description text shown below section headings where present
- Preview is read-only — no editing from the preview

TASK 10: Store templates as JSONB
- The AGENDA_TEMPLATE table stores sections as a JSONB column (sections)
- Schema (already defined in @town-meeting/shared):
  - id: string (UUID v7)
  - board_id: string (FK to BOARD)
  - town_id: string (FK to TOWN)
  - name: string
  - is_default: boolean
  - sections: JSONB (array of section objects)
  - created_at: string (ISO timestamp)
  - updated_at: string (ISO timestamp)
- In PowerSync, JSONB columns are stored as column.text (serialized JSON string)
- Parse/serialize with JSON.parse/JSON.stringify when reading/writing
- Create a Zod schema for the sections array to validate the JSONB structure before saving

TASK 11: Verification
- Navigate to /boards/:boardId/templates — the default template appears automatically
- Click "Create Template" — a new template is created and appears in the list
- Edit a template — add sections, remove non-fixed sections, reorder via drag-and-drop
- Try to remove a fixed section — it should be blocked (no remove button or disabled with tooltip)
- Edit section settings — change section_type, minutes_behavior, toggle is_fixed, add description
- Clone a template — the clone appears in the list with all sections copied
- Set a different template as default — the previous default loses its badge
- Preview a template — the preview renders the agenda structure correctly
- All changes persist across page refreshes (stored in PowerSync local SQLite)
- The template editor works offline — changes queue for sync

VERIFICATION CHECKLIST:

3. Template editor displays sections in order with drag handles
4. Fixed sections show locked icon and cannot be removed

7. Preview renders the complete template structure
8. All section types and minutes behaviors are selectable from dropdowns
9. Custom sections can be created with section_type: "other"
10. Data persists offline and syncs when connected
```
