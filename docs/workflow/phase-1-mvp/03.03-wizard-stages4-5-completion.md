# Session 03.03 — Wizard Stages 4-5 & Completion

**Phase:** 1 — MVP | **Block:** 03 — Onboarding Wizard
**Dependencies:** 03.02
**Estimated tasks:** 14

---

## Description

Implement wizard Stage 4 (Your Boards & Committees) with the standard Maine boards checklist and inline sub-configuration, Stage 5 (Meeting Style & Minutes) with formality and minutes style options, the completion handler that creates all records in a single transaction, the post-wizard quick tour overlay, and the progress checklist component for the dashboard.

## Tasks

1. Create WizardStage4 (Your Boards & Committees) with standard Maine boards checklist
2. Create "Add another board or committee" free-text input for custom boards
3. Create inline per-board sub-configuration (shown when checked): board name, member count, elected/appointed
4. Create Zod schema for Stage 4 validation (no boards required)
5. Create WizardStage5 (Meeting Style & Minutes) form: meeting formality, minutes style
6. Create Zod schemas for Stage 5 with defaults (formality: informal, minutes: summary)
7. Style formality and minutes options with descriptive help text for each option
8. Implement "Complete Setup" button on Stage 5
9. Create the completion handler: collect all wizard data, single API call to Supabase, create TOWN + BOARDs + admin PERSON/USER_ACCOUNT in one transaction
10. Implement error handling with retry: if transaction fails, show error with "Retry" button (data preserved in client state)
11. Create post-wizard quick tour overlay (4 steps: sidebar nav, progress checklist, board section, public portal link)
12. Create the progress checklist component for the dashboard
13. Redirect to Town Profile dashboard on successful completion
14. Verify: full wizard flow works end-to-end, transaction creates all records, tour displays, checklist shows

TASK 9: Create the completion handler
- Create src/lib/wizard/completeWizard.ts
- This function takes the full wizard state and creates all records in a single API call:

  async function completeWizard(data: WizardState, supabase: SupabaseClient): Promise<void>

- The function should call a Supabase Edge Function (or RPC) that handles the transaction server-side:
  - Endpoint: POST /api/wizard/complete (or supabase.rpc('complete_wizard_setup', { data }))
  - The server-side function creates everything in a single transaction:
    1. INSERT into towns: name, state, municipality_type, population_range, meeting_formality, minutes_style, presiding_officer_default, minutes_recorder_default, staff_roles_present
    2. INSERT into boards (governing board from Stage 2): name, town_id, member_count, election_method, officer_election_method, is_governing_board=true, seat_titles (JSONB if role-titled), district_based, staggered_terms
    3. INSERT into boards (for each checked Stage 4 board): name, town_id, member_count, elected_or_appointed, is_governing_board=false
    4. UPDATE user_account: set town_id on the current user's account (linking the admin to the new town)
    5. UPDATE or INSERT person: set the contact name and contact role from Stage 1

- If the Edge Function does not exist yet, create a placeholder that:
  - Sends the data as a POST request to /api/wizard/complete
  - Falls back to direct Supabase PostgREST calls (individual inserts wrapped in a client-side try/catch)
  - Add a TODO comment: "Replace with server-side transaction when Edge Function is deployed"

- The completion handler should:
  - Accept the Supabase client instance (from useSupabase hook)
  - Return void on success (records created)
  - Throw an error with a descriptive message on failure
