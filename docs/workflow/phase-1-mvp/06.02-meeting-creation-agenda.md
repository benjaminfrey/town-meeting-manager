# Session 06.02 — Meeting Creation & Agenda Building

**Phase:** 1 — MVP | **Block:** 06 — Agenda Building
**Dependencies:** 06.01
**Estimated tasks:** 14

---

## Description

Build the meeting creation workflow and the full agenda builder interface. Users create meetings by selecting a board, date, time, location, and agenda template. The agenda builder provides a rich editing interface for adding items, sub-items, and exhibits to template sections, with drag-and-drop reordering and an agenda preview.

## Tasks

1. Create MeetingListPage at /boards/:boardId/meetings showing meetings for the board (DataTable with date, title, status, actions)
2. Create CreateMeetingDialog: select board, date, time, location, select agenda template (default or custom)
3. Implement meeting creation validation: board must have >=3 members, retention policy acknowledged
4. Create AgendaBuilder page at /meetings/:meetingId/agenda — full agenda editing interface
5. Implement agenda item creation within template sections: title, description, presenter, estimated duration
6. Implement sub-items (parent_item_id) for nested agenda items
7. Create drag-and-drop reordering within and between sections
8. Create ExhibitUploader component: attach files (PDF, images, documents) to agenda items with title, exhibit_type (optional tagging per Q9: map, plan, letter, etc.), visibility (public/board-only per Q6 default: all public), sort_order
9. Implement Supabase Storage upload for exhibit files
10. Create agenda preview: formatted view of the complete agenda as it would appear on the agenda packet
11. Implement meeting cancellation: auto-update future meeting references per Q10 decision, create notice
12. Create "Approval of Minutes from [date]" agenda item type that links to previous meeting's minutes
13. Implement agenda status tracking: draft -> published
14. Verify: create meeting, build agenda with items/sub-items/exhibits, reorder, preview, cancel meeting

## Prompt

```
You are building Meeting Creation and the Agenda Builder for the Town Meeting Manager — a municipal meeting management platform for Maine towns. This session creates the workflow for creating meetings and building their agendas from templates.

Read docs/advisory-resolutions/2.2-ai-minutes-architecture.md for the agenda item data model (Section 9 — Agenda Packet Data Model), the Item Commentary format, exhibit system, and Q1-Q10 decisions. Read docs/advisory-resolutions/2.1-onboarding-wizard-ux-spec.md for meeting creation prerequisites. Key decisions:
- Q1: Item Commentary fields (background, recommendation, suggested_motion) are template-level defaults per section
- Q6: All exhibits are public by default; admins can mark specific exhibits as internal-only
- Q9: Exhibit type tagging is optional (standard, redline_draft, clean_draft, signature_page, financial_report, correspondence)
- Q10: When a meeting cancellation motion passes, auto-update the future meeting status to "Cancelled"

PROJECT CONTEXT:
- Monorepo: packages/web (React 19 + React Router v7 Framework Mode), packages/shared (types, schemas)
- PowerSync for offline-first local SQLite — use @powersync/react hooks (useQuery, useSuspenseQuery)
- Kysely ORM via @powersync/kysely-driver for type-safe SQL
- Tailwind CSS v4 + shadcn/ui (Radix UI) for components
- React Hook Form + Zod for form validation
- Supabase Storage for file uploads (exhibits)
- clientLoader pattern: load from PowerSync before route renders, then useQuery for reactivity

AGENDA ITEM DATA MODEL (from advisory 2.2 Section 9):
Each agenda item has:
- id: UUID v7
- meeting_id: FK to MEETING
- town_id: FK to TOWN
- section_type: inherited from template section
- sort_order: integer — position within its section
- title: string — item heading
- description: string | null — item description/notes
- presenter: string | null — who presents this item
- estimated_duration: number | null — minutes
- parent_item_id: string | null — FK to parent AGENDA_ITEM for sub-items
- status: pending | active | completed | tabled | deferred
- staff_resource: string | null — staff member(s) responsible (from Item Commentary)
- background: string | null — context for the board (from Item Commentary, rich text)
- recommendation: string | null — staff recommendation (from Item Commentary)
- suggested_motion: string | null — pre-drafted motion text (from Item Commentary)
- exhibits: stored in a separate EXHIBIT table linked to agenda_item_id
- created_at, updated_at

TASK 1: Create MeetingListPage
- Route: /boards/:boardId/meetings
- File: packages/web/src/routes/boards.$boardId.meetings.tsx
- Use clientLoader to load the board record and all MEETING records for this board from PowerSync
- DataTable columns: scheduled_date (formatted), title, status (Badge with color: draft=gray, scheduled=blue, in_progress=amber, completed=green, cancelled=red), action buttons
- Action buttons: Edit Agenda, Start Meeting (only if status is scheduled), View (for completed), Cancel
- Sort by scheduled_date descending (most recent first)
- "Create Meeting" button at the top opens CreateMeetingDialog
- Use PowerSync useQuery for reactive updates (new meetings from sync appear automatically)

TASK 2: Create CreateMeetingDialog
- File: packages/web/src/components/meeting/CreateMeetingDialog.tsx
- Use shadcn/ui Dialog with React Hook Form + Zod validation
- Fields:
  - Board: pre-filled from the current board context (read-only display)
  - Title: text input (default: "Regular Meeting" or "[Board Name] Meeting")
  - Date: date picker (must be in the future)
  - Time: time picker (default from board settings if available)
  - Location: text input (default from board or town settings if available)
  - Agenda Template: Select dropdown listing all templates for this board, default template pre-selected
- On submit: create MEETING record in PowerSync (status: "draft"), then instantiate the agenda from the selected template (create AGENDA_ITEM records for all default_items in the template), navigate to the AgendaBuilder page
- Meeting title auto-suggests based on board name and date: "[Board Name] — [Month Day, Year]"

TASK 3: Implement meeting creation validation
- Before showing the CreateMeetingDialog, validate:
  - Board has at least 3 active members (quorum requires >50% of seated members — with fewer than 3, quorum is impossible for a deliberative body)
  - Town retention policy has been acknowledged (checked via TOWN record field)
- If validation fails, show an alert with the specific requirement and a link to resolve it:
  - "This board needs at least 3 members before scheduling a meeting." -> link to /boards/:boardId/members
  - "The retention policy must be acknowledged before creating meetings." -> link to /dashboard
- Validation logic should use a shared utility from @town-meeting/shared

TASK 4: Create AgendaBuilder page
- Route: /meetings/:meetingId/agenda
- File: packages/web/src/routes/meetings.$meetingId.agenda.tsx
- Use clientLoader to load the MEETING record, all AGENDA_ITEM records for this meeting, and the associated template from PowerSync
- Layout: full-width page with:
  - Header: meeting title, date, status Badge, action buttons (Preview, Generate Packet, Publish)
  - Main area: sections listed vertically, each containing its agenda items
  - Each section has a header bar showing: section title, section_type icon, item count, "Add Item" button
  - Fixed sections (from template) show a locked icon — they cannot be removed but their items can be edited
  - Non-fixed sections have a "Remove Section" option (with confirmation if items exist)
  - "Add Section" button at the bottom for adding new sections not in the template
- All edits write to PowerSync local SQLite immediately — no explicit save button needed
- Status bar at bottom: total estimated duration, item count, exhibit count, draft/published status

TASK 5: Implement agenda item creation
- "Add Item" button within each section opens an inline form (not a dialog — for speed)
- Inline form fields: title (required), description (optional textarea), presenter (optional, Select from board members and staff), estimated_duration (optional, number in minutes)
- If the section's template has show_item_commentary: true, also show:
  - staff_resource (text input or Select from staff)
  - background (textarea with markdown support)
  - recommendation (textarea)
  - suggested_motion (text input — pre-drafted motion text for the live meeting)
- Suggested motion supports placeholder syntax: ___  or [TBD] — rendered as highlighted blanks
- On save: create AGENDA_ITEM record in PowerSync with sort_order set to the next available position within the section
- Inline editing: clicking an existing item expands it to show the edit form pre-filled with current values
- Delete item: trash icon with confirmation dialog ("This will also remove attached exhibits.")

TASK 6: Implement sub-items (parent_item_id)
- Agenda items can have sub-items (one level of nesting only — no deep nesting)
- Sub-items set parent_item_id to the parent AGENDA_ITEM's id
- In the UI: sub-items appear indented below their parent item
- "Add Sub-Item" button appears on each top-level item
- Sub-items inherit section_type from their parent but can have their own title, description, presenter, etc.
- Sub-items can be reordered within their parent via drag-and-drop
- Use case: Planning Board has "Map 003, Lot 059" as parent with "Application for Subdivision" and "Public Hearing" as sub-items (from advisory 2.2 Section 11)

TASK 7: Create drag-and-drop reordering
- Reuse @dnd-kit/core and @dnd-kit/sortable from session 06.01
- Items can be reordered within the same section (sort_order update)
- Items can be moved between sections (section assignment changes + sort_order update)
- Sub-items can be reordered within their parent item
- Drag handles appear on the left side of each item row
- When an item is dragged between sections, its section_type updates to match the destination section
- Reordering writes updated sort_order values to PowerSync immediately for all affected items

TASK 8: Create ExhibitUploader component
- File: packages/web/src/components/agenda/ExhibitUploader.tsx
- Attached to each agenda item — shows below the item detail when expanded
- Displays existing exhibits in a numbered list (Exhibit 1, Exhibit 2, etc.)
- "Add Exhibit" button opens a file upload area (drag-and-drop zone or file picker)
- Exhibit fields:
  - title: text input (required — e.g., "Updated Draft Policy")
  - exhibit_type: optional Select dropdown — standard (default), redline_draft, clean_draft, signature_page, financial_report, correspondence, map, plan, letter (per Q9)
  - visibility: radio buttons — Public (default per Q6) or Board-Only
  - sort_order: auto-assigned based on position in the list
- Supported file types: PDF, JPEG, PNG, DOCX, XLSX (max 10 MB per file)
- Client-side file size validation before upload
- Each exhibit stores: id, agenda_item_id, meeting_id, town_id, title, exhibit_type, visibility, sort_order, file_path (Supabase Storage path), file_name, file_size, mime_type, created_at
- Exhibits can be reordered via drag-and-drop (sort_order update)
- Delete exhibit: trash icon with confirmation

TASK 9: Implement Supabase Storage upload
- Upload files to Supabase Storage bucket "exhibits" under path: {town_id}/meetings/{meeting_id}/items/{agenda_item_id}/{exhibit_number}_{filename}
- Use the Supabase client from the PowerSync connector to get the authenticated storage client
- Show upload progress indicator (percentage bar) during upload
- On successful upload: create EXHIBIT record in PowerSync with the file_path
- On upload failure: show Toast error message with retry option
- Files are not deleted from Storage when an exhibit record is deleted (soft-delete pattern — Storage cleanup is a maintenance task)
- Support both uploaded files and URL references (a link to an external document, like the FY27 Budget Documents example from the Newcastle packet)

TASK 10: Create agenda preview
- "Preview" button in the AgendaBuilder header opens a Dialog or full-screen overlay
- The preview renders the agenda as it would appear in the published agenda packet:
  - Header: town name, board name, meeting date/time/location
  - Sections listed in order with section headings
  - Items listed under each section with numbering (section letter + item number: "6A", "6B", etc.)
  - For items with Item Commentary: show staff resource, background, exhibits list, recommendation, suggested motion (in italic)
  - Exhibits listed by number and title under each item
  - Sub-items shown indented with sub-numbering (e.g., "3A.i", "3A.ii")
- Preview mode is read-only — no editing
- Preview uses print-friendly styling (no buttons, no drag handles)

TASK 11: Implement meeting cancellation
- "Cancel Meeting" button on meetings with status "draft" or "scheduled"
- Opens a confirmation Dialog: "Are you sure you want to cancel this meeting? This action cannot be undone."
- On confirm: set meeting status to "cancelled", record cancellation timestamp and reason
- Per advisory Q10: if a future meeting references this one (e.g., "Approval of Minutes from [this meeting]"), update those references or remove them
- Generate a meeting cancellation notice (simple text: "[Board Name] meeting scheduled for [date] has been cancelled")
- Cancelled meetings remain in the list but are dimmed and non-editable

TASK 12: Create "Approval of Minutes" agenda item type
- When building an agenda, the "Minutes of Previous Meeting(s)" section auto-populates with items linking to previous meetings whose minutes are in "draft" status (not yet approved)
- Each auto-populated item title: "Approval of Minutes — [Previous Meeting Date]"
- The item links to the previous meeting's minutes document (minutes_document_id reference)
- The admin can add, remove, or reorder these items
- Multiple previous meetings' minutes can appear (e.g., if two meetings happened since the last approval)
- Query: SELECT meetings WHERE board_id = :boardId AND status = 'completed' AND minutes_status != 'approved' ORDER BY scheduled_date ASC

TASK 13: Implement agenda status tracking
- Agendas have two statuses: "draft" and "published"
- Status is stored on the MEETING record (agenda_status field)
- Draft: agenda is being built, not visible on public portal
- Published: agenda is finalized, visible on public portal, generates notifications
- "Publish" button in the AgendaBuilder header
- Publishing validation: at least one agenda item must exist, all fixed sections must be present
- Publishing shows unfilled placeholder warnings (___  or [TBD] fields) but does not block publishing
- After publishing, items can still be edited (the agenda is living until the meeting starts), but changes are tracked
- Status Badge in the header: "Draft" (gray) or "Published" (green)

- Preview shows the agenda in formatted layout with Item Commentary where enabled
- Cancel a meeting — status changes to "cancelled", confirmation required
- "Approval of Minutes" section auto-populates with previous meetings needing approval
- Publish the agenda — status changes to "published", validation runs
- All data persists across page refreshes (stored in PowerSync local SQLite)
```
