# Session 03.02 — Wizard Stages 2-3 (Governing Board & Meeting Roles)

**Phase:** 1 — MVP | **Block:** 03 — Onboarding Wizard
**Dependencies:** 03.01
**Estimated tasks:** 10

---

## Description

Implement wizard Stage 2 (Your Governing Board) with conditional seat title fields, dynamic member count behavior, and board configuration options. Implement Stage 3 (Who Runs Your Meetings) with role dropdowns and the exclusive "volunteer board" checkbox logic. Both stages integrate with the WizardProvider from session 03.01.

## Tasks

1. Create WizardStage2 (Your Governing Board) form: board name with suggestions dropdown, member count, election method, seat titles, officer election method, district-based seats, staggered terms
2. Create Zod schema for Stage 2 validation
3. Implement conditional seat titles: show N text fields matching member count when role-titled is selected, pre-fill with "1st [Board name]", "2nd [Board name]", etc.
4. Implement member count change behavior: adding/removing seat title fields while preserving existing entries
5. Show notice when member count is 0: "You can add members later. A board must have at least 3 members to hold a meeting."
6. Create WizardStage3 (Who Runs Your Meetings) form: presiding officer, minutes recorder, staff roles present with multi-select checkboxes
7. Create Zod schema for Stage 3 validation
8. Implement "None — volunteer board" exclusive checkbox behavior (unchecks all others when selected)
9. Implement back navigation preserving all stage data
10. Verify: Stage 2 conditional fields work, seat titles update dynamically, Stage 3 defaults are correct, forward/back navigation preserves state

  const stage2Schema = z.object({
    boardName: z.string()
      .min(2, "Board name must be at least 2 characters")
      .max(100, "Board name must be less than 100 characters"),
    memberCount: z.number()
      .int("Must be a whole number")
      .min(0, "Cannot be negative")

    seatTitles: z.array(z.string().min(2).max(50)).optional(),
