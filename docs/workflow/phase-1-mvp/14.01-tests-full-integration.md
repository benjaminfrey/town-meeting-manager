# Session 14.01 — Tests: Minutes, Portal, Notifications, RBAC, PWA

**Phase:** 1 — MVP | **Block:** 14 — Testing
**Dependencies:** 09.02, 10.02, 11.02, 12.01, 13.01
**Estimated tasks:** 14

---

## Description

Write comprehensive integration and end-to-end tests covering the features built in Blocks 09 through 13: minutes generation and approval, public portal with search, Postmark notifications, RBAC UI enforcement, and PWA setup. This includes a complete user journey E2E test that exercises the entire meeting lifecycle from onboarding through portal publication. All tests use Vitest for unit/integration and Playwright for E2E, following the testing strategy in advisory 1.3a.

## Tasks

1. Write integration tests for minutes generation pipeline: meeting data → JSON → HTML → PDF
2. Write component tests for MinutesReviewPage: inline editing, tracked changes, status transitions
3. Write E2E test for minutes lifecycle: generate → review → approve (via motion) → publish to portal
4. Write E2E test for public portal: verify meetings/agendas/minutes visible, search works, only approved content shown
5. Write integration test for full-text search: indexed content returns correct results
6. Write integration tests for Postmark notification dispatch: event triggers email, template renders correctly
7. Write integration test for email preference management and unsubscribe flow
8. Write component tests for RBAC: PermissionGate hides/shows elements, usePermission returns correct values
9. Write E2E test for RBAC enforcement: staff with limited permissions cannot access restricted routes
10. Write E2E test for board-scoped permissions: staff sees different UI per board
11. Write integration test for PWA: service worker caches assets, offline fallback works
12. Write E2E test for PWA: Lighthouse PWA audit passes minimum score
13. Write E2E test for complete user journey: onboarding → create meeting → build agenda → run meeting → generate/approve minutes → publish to portal
14. Run full test suite and verify all tests pass
