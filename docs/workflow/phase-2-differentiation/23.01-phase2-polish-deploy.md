# Session 23.01 — Phase 2 Integration, Polish & Deployment

**Phase:** 2 — Differentiation | **Block:** 22-23 — Testing & Deployment
**Dependencies:** 22.01
**Estimated tasks:** 8

---

## Description

Finalize Phase 2 with cross-feature integration testing, UX polish, deployment updates, and performance optimization. This session ensures all Phase 2 features work together seamlessly: AI minutes with audio transcription, SMS with straw polls, and the mobile app with the web operator session. It also prepares 10DLC documentation for town onboarding and verifies production readiness.

## Tasks

1. Integration testing across Phase 2 features: AI minutes + audio transcription, SMS + straw polls, mobile + web sync
2. Polish AI minutes UX: loading states, error handling, progress indicators during generation
3. Polish mobile app: navigation flow, offline states, sync indicators
4. Update production docker-compose.yml with new services (if any)
5. Update deployment scripts for Phase 2 services
6. Create 10DLC campaign registration documentation for town onboarding
7. Performance audit: check bundle sizes, API response times, sync latency
8. Verify: all Phase 2 features work end-to-end in production-like environment

TASK 2: Polish AI minutes UX
- Review and improve the AI minutes generation experience:
  - Loading state: show a professional skeleton loader while waiting for the first SSE event
  - Progress indicator: "Generating section 3 of 12..." with a progress bar during streaming
  - Error handling: if Claude API returns an error mid-generation, show a clear error message per section with a "Retry" button — do not lose already-generated sections
  - Timeout handling: if a section takes > 30 seconds, show "This section is taking longer than expected..." message
  - Network error: if the SSE connection drops, show "Connection lost. Retrying..." with automatic reconnection
  - Empty state: when no meeting data is available for generation, explain what data is needed
  - Success state: when all sections are generated, show a summary: "AI draft complete. X sections generated. Review each section below."
- Polish the review interface:
  - Smooth transitions between accept/reject/edit states
  - Keyboard navigation between sections (arrow keys, Enter to accept)
  - Undo support: if the operator accidentally accepts a section, allow undoing within 10 seconds
  - Mobile-responsive: ensure the review interface works on tablets (operators may use iPad)

TASK 3: Polish mobile app
- Review and improve the mobile app experience:
  - Navigation flow:
    - Bottom tab bar icons and labels are clear and consistent
    - Back button behavior is correct throughout the app
    - Deep linking from push notifications navigates to the correct screen
  - Offline states:
    - When the device is offline, show a subtle banner: "Offline — using cached data"
    - Meeting data from React Query cache still displays when offline (stale data)
    - Voting while offline: show an error toast — votes require connectivity (no local queue)
    - Show a warning if the user tries to vote while offline: "Voting requires a network connection"
  - Sync indicators:
    - The sync status badge (connected/syncing/offline) is visible during meetings
    - Pending uploads show count: "2 audio segments pending upload"
    - Upload completion toast: "Audio segment uploaded successfully"
  - Accessibility:
    - All buttons have accessible labels
    - Voting buttons are large enough for easy tapping (minimum 48dp touch target)
    - Color contrast meets WCAG AA guidelines
    - VoiceOver/TalkBack support for screen readers
  - Error handling:
    - Auth session expiry: prompt re-login without losing recording state
    - Push notification permission denied: explain value and provide settings link
    - Audio permission denied: explain how to enable in device settings
  - Platform-specific polish:
    - iOS: proper safe area handling, status bar style
    - Android: material design back button handling, notification channel setup

TASK 7: Performance audit
- Check and optimize performance across Phase 2 features:
  - Web bundle size:
    - Run the build and analyze bundle: check that the Anthropic SDK, Twilio types, and audio processing libraries are not bloating the client bundle (they should be server-side only)
    - Verify tree shaking is working for @town-meeting/shared imports
    - Target: web bundle under 500KB gzipped for initial load
  - API response times:
    - AI minutes generation: streaming should begin within 2 seconds of request
    - Transcription job creation: API should respond within 500ms (job runs async)
    - SMS send: API should respond within 1 second (Twilio call is async)
    - Straw poll vote: API should respond within 200ms
  - Supabase Realtime sync latency:
    - Measure web-to-mobile sync time during a meeting simulation
    - Target: Realtime changes propagate within 2 seconds
    - Check that audio segment metadata syncs without transferring the audio blob through Realtime
  - Mobile app performance:
    - App launch time: target under 3 seconds (cold start to dashboard)
    - Meeting view render: target under 500ms
    - Memory usage during audio recording: monitor for leaks during extended recording
  - Database query performance:
    - Check that new tables (WARRANT_ARTICLE, STRAW_POLL, SMS_CONSENT, TRANSCRIPT, AUDIO_SEGMENT) have appropriate indexes
    - Run EXPLAIN ANALYZE on critical queries (poll results aggregation, resident subscriber lookup, transcription job status)
