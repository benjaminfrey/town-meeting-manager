# Session 16.02 — AI Minutes Review UX

**Phase:** 2 — Differentiation | **Block:** 16 — AI Minutes Drafting
**Dependencies:** 16.01
**Estimated tasks:** 9

---

## Description

Build the AI-specific review tools for the MinutesReviewPage, enabling operators to verify AI-generated minutes against the structured meeting data. This includes a side-by-side comparison view, section-level accept/reject/edit workflow, per-section regeneration, confidence indicators, factual accuracy checking, and audit tracking of AI vs. manually edited content.

## Tasks

1. Enhance MinutesReviewPage with AI-specific review tools
2. Create side-by-side view: AI draft on one side, structured meeting data (motions, votes, attendance) on the other for verification
3. Create section-level accept/reject/edit workflow: operator reviews each AI-generated section individually
4. Implement AI regeneration per section: if a section is rejected, operator can edit the prompt context and regenerate just that section
5. Create "confidence indicators" for AI-generated content: flag sections where the AI had limited data (e.g., no motion recorded, sparse timestamps)
6. Create factual accuracy checklist: auto-verify that all recorded motions, votes, and attendance match between the AI draft and the structured data
7. Implement manual override: operator can replace any AI-generated text with manual content
8. Track which sections were AI-generated vs. manually edited (for audit/transparency)
9. Verify: AI draft displays with review tools, section-level editing works, regeneration works, accuracy checklist catches discrepancies

TASK 2: Create side-by-side comparison view
- Create packages/web/src/components/minutes/AIDraftComparison.tsx
- Layout: two-column panel (responsive — stacks on mobile)

  - Right column: "Meeting Data" — the raw structured data that was sent to the AI for this section

TASK 3: Create section-level accept/reject/edit workflow
- Create packages/web/src/components/minutes/SectionReviewCard.tsx
- Each AI-generated section is rendered in a review card with three actions:
  - Accept (green checkmark button): Mark this section as reviewed and approved. The AI text is accepted as-is.
  - Edit (pencil button): Open the section in an inline rich text editor (textarea or prosemirror-style). The operator can modify the AI text. On save, the section is marked as 'edited' with the editor's person_id and timestamp.

  - The AI-generated text content
  - The confidence indicator (Task 5)
  - The comparison panel (Task 2) expandable below the text

- Navigation: "Previous Section" and "Next Section" buttons for sequential review
- Keyboard shortcuts: Ctrl+Enter to accept, Ctrl+E to edit, Ctrl+R to reject
- When editing, use a textarea with the AI text pre-filled. The editor supports basic formatting: paragraphs, and tabbed indentation. Show a character count.
- On save after editing: update the MINUTES_DOCUMENT section with the new content, set generated_by to 'manual', record editedBy and editedAt

TASK 4: Implement per-section AI regeneration
- Create packages/web/src/components/minutes/SectionRegenerator.tsx
- When a section is rejected, this panel appears:
  - Shows the current prompt context: what data was sent to the AI for this section
  - Provides an "Additional Context" textarea where the operator can add notes to guide regeneration (e.g., "The discussion was primarily about budget concerns" or "Include more detail about the public comment from Jane Doe")
  - "Minutes Style" override dropdown: allows changing the style for just this section (e.g., switch from 'summary' to 'narrative' for a contentious item)
  - "Regenerate" button that calls the streaming API endpoint from 16.01 for just this single section
- During regeneration:
  - Show a loading state with streaming text appearing in real-time (subscribe to the SSE stream from 16.01)
  - The old text is dimmed in the background for comparison
  - When complete, the new text replaces the old text and the section status changes to "Regenerated"
- The operator can regenerate multiple times — each attempt is a new API call
- Track regeneration count per section in the metadata (for cost monitoring)
- If regeneration fails, show the error and offer to keep the original text or enter manual text

TASK 5: Create confidence indicators
- Create packages/web/src/components/minutes/ConfidenceIndicator.tsx
- Analyze the data completeness for each section and display a confidence level:
  - High confidence (green dot): Section has complete data — motions with full text, named mover/seconder, complete vote tallies, timestamps
  - Medium confidence (amber dot): Section has partial data — some fields present but gaps exist (e.g., motion recorded but no second, or discussion notes but no specific speakers)
  - Low confidence (red dot): Section has minimal data — few or no operator notes, no transcript, generic timestamp, or the AI flagged [VERIFY] markers in the output
- Confidence calculation logic (in @town-meeting/shared or in the component):
  - Check for presence of: motions (if action section), operator_notes, timestamps (start and end), speaker attributions, vote tallies
  - Count [VERIFY: ...] markers in the AI output
  - Score: all data present = high, 60-99% = medium, below 60% = low
- Display as a small badge next to each section title: "AI Confidence: High/Medium/Low"
- Low confidence sections should auto-expand the comparison panel for easier verification
- Add a tooltip explaining what data was missing or limited

TASK 6: Create factual accuracy checklist
- Create packages/web/src/components/minutes/AccuracyChecklist.tsx
- This component auto-verifies the AI draft against the structured meeting data:
  - Attendance check: Does the AI draft mention all present members? Does it correctly note absent members?
  - Motion check: For each motion in the structured data, does the AI draft include it? Does it match the mover, seconder, and motion text?
  - Vote check: For each vote in the structured data, does the AI draft match the vote tally (yea/nay/abstain counts)?
  - Timestamp check: Are the call-to-order and adjournment times correct?
  - Quorum check: Does the AI draft correctly state the quorum status?
  - Presiding officer check: Is the correct presiding officer mentioned?
- Implementation approach:
  - Parse the AI draft text for key phrases (names, vote patterns like "4-0", timestamps)
  - Compare against the structured data fields
  - This is heuristic matching — look for the presence of expected strings, not perfect NLP
- Display as a checklist at the top of the review page or in a collapsible sidebar:
  - Each check shows: pass (green check), fail (red X), or warning (amber triangle)
  - Failed checks include a description: "Motion #2 (agenda item 5B): AI draft shows '3-1 vote' but recorded data shows '4-0 vote'"
  - Clicking a failed check scrolls to and highlights the relevant section
- The checklist updates in real-time as sections are edited

TASK 7: Implement manual override
- Within any section's edit mode (Task 3), the operator can completely replace the AI text
- Create a "Clear & Write Manually" button that:
  - Clears the AI-generated text
  - Opens an empty text area for manual entry
  - Sets generated_by to 'manual' for this section
  - Removes confidence indicators (not applicable for manual content)
- The operator can also paste text from an external source (e.g., minutes prepared in Word)
- Manual sections are visually distinct from AI sections in the review view (different background color or border)
- Manual override is available for any section at any time, even after accepting the AI version

- Add an "Audit Trail" expandable section at the bottom of each section card showing the edit history
