# Session 17.01 — Single-Device Audio Recording

**Phase:** 2 — Differentiation | **Block:** 17 — Audio Recording & Transcription
**Dependencies:** 07.03
**Estimated tasks:** 10

---

## Description

Build the browser-based audio recording system for meeting sessions using the Web Audio API. The recorder captures audio during live meetings, automatically segments recordings by agenda item, uploads segments to Supabase Storage, and provides playback for post-meeting review. This audio data feeds into the transcription pipeline (session 17.02) and enriches AI minutes generation.

## Tasks

1. Create AudioRecorder component using Web Audio API (MediaRecorder API for browser)
2. Implement recording controls: start, pause, resume, stop with visual waveform display
3. Create audio segment management: automatically segment recording by agenda item (when operator advances to next item, start new audio segment)
4. Implement audio file storage: upload completed segments to Supabase Storage with meeting_id and agenda_item_id metadata
5. Create audio playback component for post-meeting review
6. Implement recording quality settings: sample rate, bit depth, format (WebM/Opus for browser)
7. Handle browser permissions: microphone access request, permission denied handling
8. Create recording status indicator on LiveMeetingPage: recording active/paused/stopped
9. Implement automatic recording start when meeting begins (with operator confirmation)
10. Verify: audio records during meeting, segments created per agenda item, files upload to Storage, playback works

```

TASK 1: Create AudioRecorder component
- Create packages/web/src/components/meeting/AudioRecorder.tsx
- Use the MediaRecorder API (Web Audio API) for browser-based recording
- Initialize with navigator.mediaDevices.getUserMedia({ audio: true })
- Configure MediaRecorder with options:
  - mimeType: 'audio/webm;codecs=opus' (widely supported, good compression)
  - audioBitsPerSecond: configurable (default 128000 — 128kbps, good quality for speech)
- Collect audio data chunks via the ondataavailable event
  - Set timeslice to 1000ms (1 second chunks) for regular data collection
  - Store chunks in a ref array for the current segment
- The component manages recording state internally:
  - RecordingState: 'idle' | 'recording' | 'paused' | 'stopped'
- Export a hook useAudioRecorder() that provides:
  - state: RecordingState
  - startRecording(): void
  - pauseRecording(): void
  - resumeRecording(): void
  - stopRecording(): Promise<Blob> (returns the complete audio blob)
  - currentSegmentDuration: number (seconds, updated via requestAnimationFrame)
  - audioLevel: number (0-1, for visual feedback — computed from AnalyserNode)
  - error: string | null

TASK 2: Implement recording controls UI
- Create packages/web/src/components/meeting/RecordingControls.tsx
- Renders a control bar within the LiveMeetingPage:
  - Record button (red circle): starts recording, changes to a pause icon when active
  - Pause button: pauses recording, changes to resume icon
  - Stop button: stops recording and finalizes the current segment
  - Duration display: "00:45:23" running timer showing total recording time
  - Segment indicator: "Segment 3 of 7 | Current: 05:32" showing which agenda item segment is active and its duration
- Create a visual waveform display:
  - Use Web Audio API AnalyserNode connected to the MediaRecorder's stream
  - Create a canvas-based waveform visualization (simple bar graph showing audio levels)
  - Update at 30fps using requestAnimationFrame
  - The waveform provides visual feedback that audio is being captured (operators can see that the microphone is picking up sound)
- Show recording quality indicator: green dot when signal is good, amber when low, red when silence detected for >10 seconds (suggests microphone issue)
- Use shadcn/ui Button components styled with recording-specific colors (red for active recording)

TASK 3: Create audio segment management
- Create packages/web/src/services/audio-segment-manager.ts
- Segments are created automatically when the operator advances to the next agenda item on the LiveMeetingPage:
  1. When the operator clicks "Next Item" or selects a different agenda item:
     a. Finalize the current segment: stop the current MediaRecorder, collect all chunks into a Blob
     b. Create a segment record: { meetingId, agendaItemId, segmentNumber, startTime, endTime, blob, status: 'recorded' }
     c. Queue the segment for upload (Task 4)
     d. Start a new MediaRecorder for the next agenda item's segment
  2. This transition should be seamless — minimize the gap between stopping one segment and starting the next (target < 500ms)
- Handle edge cases:
  - Operator skips an agenda item: create a segment with duration 0 or skip it entirely
  - Operator goes back to a previous item: resume recording for that item's segment (append to existing segment or create a new sub-segment)
  - Meeting pauses: when recording is paused, all segments pause; when resumed, the current segment continues
- Store segment metadata in PowerSync for offline tracking:
  - AUDIO_SEGMENT table: id, meeting_id, agenda_item_id, segment_number, duration_seconds, file_path (Supabase Storage path), file_size_bytes, status (recorded/uploading/uploaded/failed), created_at
- Export functions: startNewSegment(agendaItemId), finalizeCurrentSegment(), getSegments(meetingId)

TASK 4: Implement audio file storage
- Create packages/web/src/services/audio-upload.ts
- Upload completed audio segments to Supabase Storage:
  - Bucket: "meeting-audio"
  - Path: {town_id}/meetings/{meeting_id}/segments/{segment_number}_{agenda_item_id}.webm
  - Upload with metadata: { meetingId, agendaItemId, segmentNumber, duration, recordedAt }
- Implement upload queue:
  - Segments are queued for upload as they are finalized
  - Upload runs in the background while recording continues
  - Queue processes one upload at a time to avoid bandwidth contention with the live meeting
  - Failed uploads are retried (3 attempts, exponential backoff)
  - If offline, uploads queue in IndexedDB and resume when connectivity returns
- Show upload status per segment: pending, uploading (with progress %), uploaded, failed
- Handle large files: segments for long agenda items could be 50+ MB. Use chunked upload if Supabase supports it, otherwise upload as a single blob.
- Update the AUDIO_SEGMENT record in PowerSync when upload completes (set status to 'uploaded', store file_path)

TASK 5: Create audio playback component
- Create packages/web/src/components/meeting/AudioPlayer.tsx
- Post-meeting playback for reviewing recordings:
  - Playlist view: list all segments for the meeting, organized by agenda item
  - Each segment shows: agenda item title, duration, upload status
  - Click a segment to play it
  - Standard audio controls: play/pause, seek bar, volume, playback speed (0.5x, 1x, 1.5x, 2x)
  - Display current timestamp during playback
- Continuous playback mode: play all segments sequentially (entire meeting audio)
- Segment navigation: previous/next segment buttons
- The player loads audio from Supabase Storage (authenticated URL via supabase.storage.from('meeting-audio').getPublicUrl() or createSignedUrl())
- Use the HTML5 <audio> element with a custom UI wrapper using shadcn/ui components
- Show a loading state while the audio file downloads
- This component is used on the post-meeting review page and the minutes review page

TASK 6: Implement recording quality settings
- Create packages/web/src/components/meeting/RecordingSettings.tsx
- Settings dialog (accessible from the recording controls):
  - Audio input device selection: dropdown of available microphones (from navigator.mediaDevices.enumerateDevices())
  - Quality preset: "Standard" (64kbps, smaller files) | "High" (128kbps, recommended) | "Maximum" (256kbps, large files)
  - Auto-segment: toggle on/off (default: on — automatically segment by agenda item)
  - Auto-start: toggle on/off (default: off — prompt operator before starting recording when meeting begins)
- Settings are stored in localStorage per device (not synced — hardware-specific)
- Show estimated file size: "~2 MB per 10 minutes at High quality"
- Audio input level meter: real-time visualization of the selected microphone's input level (helps operator verify the mic is picking up audio before the meeting starts)
- "Test Recording" button: record 5 seconds, play it back, confirm audio quality before the meeting

TASK 7: Handle browser permissions
- Create packages/web/src/hooks/useMicrophonePermission.ts
- Manage the microphone permission lifecycle:
  1. Check current permission state: navigator.permissions.query({ name: 'microphone' })
  2. If 'prompt': show a pre-permission dialog explaining why audio recording is needed ("Town Meeting Manager records meeting audio to generate accurate minutes. Your browser will ask for microphone access.")
  3. If 'granted': proceed to recording setup
  4. If 'denied': show a help dialog explaining how to re-enable microphone access in browser settings (instructions for Chrome, Firefox, Safari, Edge)
- Handle getUserMedia errors:
  - NotAllowedError: permission denied — show instructions
  - NotFoundError: no microphone — show "No microphone detected. Please connect a microphone."
  - NotReadableError: microphone in use — show "Microphone is being used by another application."
  - OverconstrainedError: requested settings not supported — fall back to default constraints
- The permission check runs when the operator first clicks "Start Recording", not on page load
- Store permission state to avoid re-prompting on subsequent meetings

TASK 8: Create recording status indicator
```
