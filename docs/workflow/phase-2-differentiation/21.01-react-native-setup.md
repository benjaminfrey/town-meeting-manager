# Session 21.01 — React Native + Expo Setup

**Phase:** 2 — Differentiation | **Block:** 21 — React Native Mobile App
**Dependencies:** 01.02
**Estimated tasks:** 10

---

## Description

Initialize the React Native mobile application using Expo in the packages/mobile directory. This session sets up the Expo project with TypeScript, NativeWind for Tailwind-like styling, Expo Router for file-based navigation, TanStack Query for data fetching, and the Supabase client. The mobile app shares types and schemas from @town-meeting/shared and provides board members with meeting participation tools on their personal devices.

## Tasks

1. Initialize packages/mobile with Expo SDK (npx create-expo-app)
2. Configure TypeScript for mobile package (extending tsconfig.base.json)
3. Install and configure NativeWind v4/v5 for Tailwind-like styling on React Native
4. Install Expo Router v6 for file-based navigation
5. Install TanStack Query v5 and configure QueryClient for mobile
6. Install Supabase client for React Native
7. Create typed Supabase hooks and query key factory for mobile (sharing patterns from web)
8. Add @town-meeting/shared as workspace dependency (shared types, schemas, validators)
9. Set up basic app structure: login screen, dashboard, board list, meeting view
10. Verify: Expo dev server starts, app runs in simulator, React Query fetches data, shared types import correctly

## Prompt

```
You are setting up the React Native mobile application using Expo for the Town Meeting Manager — a municipal meeting management platform for Maine towns. The mobile app provides board members with meeting participation tools (view agendas, cast votes, record audio) on their personal devices. It shares the same backend and data layer as the web application.

Read docs/advisory-resolutions/3.1-monorepo-structure.md for the monorepo package structure. The mobile package at packages/mobile was created as a placeholder in session 01.01 — this session replaces that placeholder with a fully configured Expo project.

PROJECT CONTEXT:
- Monorepo: packages/web (React 19 web app), packages/mobile (React Native/Expo), packages/api (Fastify), packages/shared (types, schemas, validators)
- pnpm workspaces with Turborepo build orchestration
- Supabase for auth and PostgreSQL database
- TanStack Query v5 for data fetching and caching (same patterns as the web app)
- @town-meeting/shared provides TypeScript types, Zod schemas, and validation functions shared across web and mobile
- The mobile app targets iOS and Android via Expo
- The mobile app is for authenticated users (board members, staff) — NOT for public residents

TASK 1: Initialize Expo project in packages/mobile
- Remove the placeholder content in packages/mobile/ (the .gitkeep and placeholder package.json from session 01.01)
- Initialize a new Expo project:
  cd packages/mobile
  npx create-expo-app@latest . --template blank-typescript
- Update the generated package.json:
  - name: "@town-meeting/mobile"
  - version: "0.0.1"
  - private: true
  - Ensure the Expo SDK version is latest stable (SDK 52+)
- Verify the Expo project structure is created: app.json (or app.config.ts), src/ directory, assets/
- Prefer app.config.ts over app.json for type-safe configuration:
  - app name: "Town Meeting Manager"
  - slug: "town-meeting-manager"
  - scheme: "townmeeting" (for deep linking)
  - iOS bundleIdentifier: "com.townmeetingmanager.app"
  - Android package: "com.townmeetingmanager.app"
  - orientation: "portrait" (default, but allow landscape for tablets)
  - icon and splash screen: use placeholder images for now

TASK 2: Configure TypeScript for mobile
- Update packages/mobile/tsconfig.json to extend the monorepo base:
  {
    "extends": "../../tsconfig.base.json",
    "compilerOptions": {
      "jsx": "react-jsx",
      "lib": ["ES2022"],
      "moduleResolution": "bundler",
      "strict": true,
      "noEmit": true,
      "paths": {
        "@/*": ["./src/*"],
        "@town-meeting/shared/*": ["../shared/src/*"]
      }
    },
    "include": ["src/**/*", "app/**/*", "*.config.ts"],
    "exclude": ["node_modules", "dist", ".expo"]
  }
- Note: React Native/Expo uses Metro bundler, not a standard Node.js build. The tsconfig is for type checking only — Metro handles the actual bundling.
- Ensure strict mode is enabled to match the rest of the monorepo
- Add a typecheck script to package.json: "typecheck": "tsc --noEmit"

TASK 3: Install and configure NativeWind
- NativeWind brings Tailwind CSS-like utility classes to React Native:
  pnpm --filter @town-meeting/mobile add nativewind
  pnpm --filter @town-meeting/mobile add -D tailwindcss@^3
- NativeWind v4/v5 uses Tailwind CSS v3 under the hood for React Native (not v4 — Tailwind v4 does not support React Native yet)
- Create tailwind.config.js in packages/mobile/:
  module.exports = {
    content: ['./src/**/*.{js,jsx,ts,tsx}', './app/**/*.{js,jsx,ts,tsx}'],
    presets: [require('nativewind/preset')],
    theme: {
      extend: {
        colors: {
          // Match the web app's color palette for visual consistency
          primary: { /* blues */ },
          destructive: { /* reds */ },
          muted: { /* grays */ },
        },
      },
    },
    plugins: [],
  };
- Configure Metro to process NativeWind:
  - Create or update metro.config.js to include the NativeWind resolver
  - Add the NativeWind babel plugin to babel.config.js
- Create a global.css file that imports Tailwind directives: @tailwind base; @tailwind components; @tailwind utilities;
- Import global.css in the app layout file
- Verify: create a test component with className="bg-blue-500 p-4" — it should render correctly

TASK 4: Install and configure Expo Router
- Expo Router provides file-based routing (similar to Next.js or React Router v7 Framework Mode):
  pnpm --filter @town-meeting/mobile add expo-router
- This also requires:
  pnpm --filter @town-meeting/mobile add expo-linking expo-constants expo-status-bar react-native-safe-area-context react-native-screens
- Configure Expo Router in app.config.ts:
  - Set "scheme": "townmeeting"
  - Set "web.bundler": "metro" if web support is desired
- Create the app/ directory for file-based routes:
  app/
  ├── _layout.tsx          # Root layout (AuthProvider, QueryClientProvider)
  ├── (auth)/
  │   ├── _layout.tsx      # Auth layout (no navigation chrome)
  │   └── login.tsx         # Login screen
  ├── (app)/
  │   ├── _layout.tsx      # App layout (tab navigation)
  │   ├── index.tsx         # Dashboard / home tab
  │   ├── boards/
  │   │   ├── index.tsx     # Board list
  │   │   └── [boardId].tsx # Board detail
  │   └── meetings/
  │       ├── index.tsx     # Meeting list
  │       └── [meetingId].tsx  # Meeting view
  └── +not-found.tsx        # 404 screen
- Configure the root layout with navigation providers:
  - AuthProvider wrapping QueryClientProvider wrapping the router Stack/Tabs
  - Use expo-router's Redirect component for auth-based routing

TASK 5: Install TanStack Query v5
- pnpm --filter @town-meeting/mobile add @tanstack/react-query
- Create packages/mobile/src/lib/query-client.ts:
  - Create a QueryClient with the same defaults as the web app:
    const queryClient = new QueryClient({
      defaultOptions: {
        queries: { staleTime: 60_000, gcTime: 5 * 60_000, retry: 2, refetchOnWindowFocus: false },
        mutations: { retry: 0 },
      },
    });
  - Export the queryClient singleton
  - Note: React Native uses refetchOnWindowFocus: false (no window focus events on mobile — use refetchOnMount instead)
- Create packages/mobile/src/providers/QueryProvider.tsx:
  - Wrap QueryClientProvider with the queryClient singleton
  - Import from @tanstack/react-query
- Re-use the queryKeys factory from @town-meeting/shared (if exported) or create packages/mobile/src/lib/query-keys.ts mirroring the web version

TASK 6: Install Supabase client for React Native
- pnpm --filter @town-meeting/mobile add @supabase/supabase-js
- React Native requires an AsyncStorage adapter for session persistence:
  pnpm --filter @town-meeting/mobile add @react-native-async-storage/async-storage
- Create packages/mobile/src/lib/supabase.ts:
  import { createClient } from '@supabase/supabase-js';
  import AsyncStorage from '@react-native-async-storage/async-storage';

  const supabase = createClient(
    process.env.EXPO_PUBLIC_SUPABASE_URL!,
    process.env.EXPO_PUBLIC_SUPABASE_ANON_KEY!,
    {
      auth: {
        storage: AsyncStorage,
        autoRefreshToken: true,
        persistSession: true,
        detectSessionInUrl: false,  // disable for React Native (no URL-based auth)
      },
    }
  );
  export default supabase;
- Create a .env file (and .env.example) with:
  EXPO_PUBLIC_SUPABASE_URL=http://localhost:54321
  EXPO_PUBLIC_SUPABASE_ANON_KEY=your-anon-key

TASK 7: Create typed Supabase hooks for mobile
- Create packages/mobile/src/lib/api/boards.ts — typed hooks for board queries:
  export function useBoards(townId: string) {
    return useQuery({
      queryKey: queryKeys.boards(townId),
      queryFn: () => supabase.from('board').select('*').eq('town_id', townId).throwOnError(),
      enabled: !!townId,
    });
  }
- Create packages/mobile/src/lib/api/meetings.ts — typed hooks for meeting queries:
  export function useMeetings(boardId: string) {
    return useQuery({
      queryKey: queryKeys.meetings(boardId),
      queryFn: () => supabase.from('meeting').select('*, board(name)').eq('board_id', boardId).throwOnError(),
      enabled: !!boardId,
    });
  }
- Create packages/mobile/src/lib/mutations/votes.ts — useMutation for voting:
  export function useSubmitVote() {
    return useMutation({
      mutationFn: (vote: VoteInput) => supabase.from('vote_record').upsert(vote).throwOnError(),
      onSuccess: () => queryClient.invalidateQueries({ queryKey: queryKeys.votes() }),
    });
  }
- The mobile hooks follow the same patterns as packages/web/src/lib/api/ for consistency

TASK 8: Add @town-meeting/shared as workspace dependency
- In packages/mobile/package.json, add:
  "dependencies": {
    "@town-meeting/shared": "workspace:*"
  }
- Run pnpm install to link the workspace dependency
- Configure Metro to resolve the shared package:
  - Update metro.config.js to include packages/shared in the watchFolders:
    const { getDefaultConfig } = require('expo/metro-config');
    const path = require('path');

    const config = getDefaultConfig(__dirname);
    const projectRoot = __dirname;
    const monorepoRoot = path.resolve(projectRoot, '../..');

    config.watchFolders = [monorepoRoot];
    config.resolver.nodeModulesPaths = [
      path.resolve(projectRoot, 'node_modules'),
      path.resolve(monorepoRoot, 'node_modules'),
    ];
    // Ensure packages/shared TypeScript files are transpiled
    config.resolver.disableHierarchicalLookup = true;

    module.exports = config;
- Verify imports work: import { APP_NAME } from '@town-meeting/shared'; should resolve correctly
- Import shared types: WarrantArticle, Meeting, Board, Person, etc.
- Import shared Zod schemas for form validation on mobile
```
